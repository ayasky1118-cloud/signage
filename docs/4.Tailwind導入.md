# Tailwind CSS v4 導入手順（Vite + Vue3）

本ドキュメントは、Vite + Vue3 プロジェクトに Tailwind CSS v4 を導入するための手順書である。
Tailwind v4 は **Viteプラグイン（@tailwindcss/vite）** のみを使う構成で導入する（CLI・PostCSS・autoprefixer は不要）。

---

## 1. 前提確認（プロジェクト位置）

### 1-1. Viteプロジェクトのルートへ移動

`package.json` と `vite.config.ts` と `index.html` が存在するディレクトリへ移動する。

```bash
cd /path/to/your/frontend
```

### 1-2. 必須ファイルの存在確認

```bash
ls
```

以下が存在すること：

- `package.json`
- `vite.config.ts`（または `vite.config.js`）
- `index.html`
- `src/`

---

## 2. Tailwind v4 をインストール（Viteプラグイン方式）

### 2-1. 必要パッケージを開発依存でインストール

```bash
npm install -D tailwindcss @tailwindcss/vite
```

> **補足：** `tailwindcss/postcss` や `autoprefixer` は @tailwindcss/vite 使用時には不要。既存プロジェクトに残っていても動作に支障はないが、本手順では使用しない。

---

## 3. Vite設定に Tailwind を追加

`vite.config.ts` を編集し、Tailwindプラグインを `plugins` に追加する。

例：

```ts
import { defineConfig } from "vite"
import vue from "@vitejs/plugin-vue"
import tailwindcss from "@tailwindcss/vite"

export default defineConfig({
  plugins: [vue(), tailwindcss()],
})
```

> 既に plugins 配列がある場合は、配列に `tailwindcss()` を追加する。

---

## 4. Tailwind CSS エントリーファイル作成

### 4-1. `src/style.css` を Tailwind エントリに変更

既存の `src/style.css` を、Tailwind のエントリーポイントとして使う。

```css
@import "tailwindcss";
```

> Vite のデフォルトテンプレートで生成された style.css の内容は、common.css 等で上書きする前提のため削除してよい。

---

## 5. main.ts でCSSを読み込む

`src/main.ts` で、**以下の順序**で CSS を読み込む。

1. **common.css**（グローバル共通スタイル）を先に読み込む
2. **style.css**（Tailwind エントリ）を後に読み込む

```ts
import { createApp } from "vue"
import "./assets/styles/common.css"
import "./style.css"
import App from "./App.vue"
import router from "./router"

createApp(App).use(router).mount("#app")
```

> **理由：** common.css で定義した CSS 変数やベーススタイルを先に適用し、その上に Tailwind のユーティリティを重ねるため。

### ページ専用 CSS について

`order-list.css` など、特定ページ専用のスタイルは **main.ts では読み込まず**、該当する Vue ページ（例：OrderList.vue）の `<script setup>` 内で import する。

```ts
// OrderList.vue
import "../assets/styles/order-list.css"
```

---

## 6. 開発サーバー再起動

設定ファイル追加・更新後は Vite を再起動する。

```bash
# 起動中のターミナルで
Ctrl + C

npm run dev
```

---

## 7. 動作確認（簡易テスト）

`App.vue` に一時的に以下を記述する。

```vue
<template>
  <div class="p-10 bg-blue-600 text-white text-2xl font-bold">
    Tailwind OK
  </div>
</template>
```

青背景＋白文字＋太字になれば導入成功。

---

## 補足：index.html の Google Fonts（任意）

`index.html` の `<head>` に Google Fonts を追加する場合は、以下のように記述する。**任意**であり、Tailwind 導入には不要。

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;500;700&display=swap"
  rel="stylesheet"
/>
```

---

## 補足：CLIについて

`npx tailwindcss -v` が動かない環境がある。
本手順は Viteプラグイン方式で導入するため、CLIは必須ではない。

（必要になった場合のみ）
```bash
npm install -D @tailwindcss/cli
npx @tailwindcss/cli --version
```

---

## 補足：Tailwind 動作確認（DevTools）

ブラウザの DevTools（F12）で以下を確認する。

1. **Elements** タブで、Tailwind のユーティリティクラス（例：`bg-blue-600`, `p-4`）が付いた要素を選択
2. **Styles** パネルに、該当クラスに対応する CSS が表示されていれば Tailwind が効いている
3. または、`<head>` 内に `style` タグで Tailwind の生成スタイルが注入されていることを確認
