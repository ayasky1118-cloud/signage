# 地図描画 最小動作確認（手順 11-1）

手順 11-1 の詳細な確認手順です。

---

## 前提条件

- フロントエンド: `npm run dev` で起動済み
- バックエンド API: 起動済み（`/api/html-objects`, `/api/orders` 等）
- 環境変数: `VITE_MAPTILER_API_KEY` が設定済み（地図表示に必要）
- 注文検索: テスト用の注文番号が存在するか、新規登録フローで確認可能

---

## 確認手順

### 1. 全画面編集を開く

| 手順 | 操作 | 期待結果 |
|------|------|----------|
| 1-1 | 看板編集画面（OrderDetail）を開く | 注文検索エリア・枝番タブ・地図プレビューが表示される |
| 1-2 | 注文番号を入力し検索（または一覧から遷移） | 注文情報が表示され、枝番タブが表示される |
| 1-3 | 「全画面で編集・操作する」ボタンをクリック | 全画面編集オーバーレイが開き、左サイドバーと右マップが表示される |

**確認ポイント**: 地図が表示され、サイドバーに「ルート描画」「テキスト配置」等のカテゴリが表示されること。

---

### 2. ルート描画を選択し、地図上をクリックしてマーカーを複数打つ

| 手順 | 操作 | 期待結果 |
|------|------|----------|
| 2-1 | ルート描画のドロップダウンで線種を選択 | 「選択してください」のときは描画ボタンが無効。線種選択後、描画ボタンが有効になる |
| 2-2 | 「描画」ボタン（鉛筆アイコン）をクリック | ボタンがアクティブ状態になり、地図クリックでマーカー追加モードになる |
| 2-3 | 地図上を 2 点以上クリック | 赤い丸の一時マーカーが表示される |
| 2-4 | 「確定」ボタンが有効になることを確認 | マーカー 2 点以上で「確定」が押せる状態になる |

**確認ポイント**: 一時マーカーが赤色で表示され、クリックごとに追加されること。

**キャンセル**: 1 点だけ打って戻りたい場合は、描画ボタン（アクティブ状態）を再度クリックする。一時マーカーがクリアされ、描画モードが解除される。

---

### 3. 「確定」ボタンで線を確定する

| 手順 | 操作 | 期待結果 |
|------|------|----------|
| 3-1 | 「確定」ボタンをクリック | 一時マーカーが消え、マーカー間を結ぶ線が表示される |

**確認ポイント**: 線が地図上に描画されること。

---

### 4. 線が赤色で表示されることを確認する

| 手順 | 操作 | 期待結果 |
|------|------|----------|
| 4-1 | 描画した線を目視確認 | 線が赤色（#FF0000）、太さ 4px で表示される |

**確認ポイント**: デフォルトは赤色。ドロップダウンで value_data に `{"color":"#0000FF","width":6}` 等を設定した線種を選択した場合は、その色・太さで表示される。

---

### 5. 枝番を切り替えて、他枝番のデータが混ざらないことを確認する

| 手順 | 操作 | 期待結果 |
|------|------|----------|
| 5-1 | 枝番 01 でルートを 1 本描画 | 線が表示される |
| 5-2 | 枝番タブで枝番 02 に切り替え | 地図がクリアされ、枝番 02 のデータ（空）が表示される |
| 5-3 | 枝番 02 で別のルートを 1 本描画 | 枝番 02 用の線が表示される |
| 5-4 | 枝番 01 に戻る | 枝番 01 で描画した線のみが表示される |
| 5-5 | 枝番 02 に戻る | 枝番 02 で描画した線のみが表示される |

**確認ポイント**: 枝番ごとにデータが独立して保持・表示されること。

---

### 6. 登録ボタンで design_data が API に送信されることを確認する

| 手順 | 操作 | 期待結果 |
|------|------|----------|
| 6-1 | ルートを描画した状態で「更新」または「登録」ボタンをクリック | 確認モーダルが表示される |
| 6-2 | 確認で OK を押す | API にリクエストが送信される |
| 6-3 | 開発者ツールの Network タブでリクエストを確認 | `design_data` に `routes` 配列が含まれる |
| 6-4 | 画面をリロードし、同じ注文を再検索 | 描画したルートが復元されて表示される |

**確認ポイント**: `design_data.routes` に `{ id, type, coordinates, color?, width? }` 形式のデータが含まれること。

## トラブルシューティング

| 現象 | 想定原因 | 対処 |
|------|----------|------|
| 地図が表示されない | MapTiler API キー未設定 | `VITE_MAPTILER_API_KEY` を設定 |
| 描画ボタンが押せない | 線種が未選択 | ドロップダウンで線種を選択する |
| クリックしてもマーカーが追加されない | 描画モードになっていない | 「描画」ボタンを先にクリック |
| 確定ボタンが押せない | マーカーが 1 点以下、または線種未選択 | 2 点以上クリックしてマーカーを打つ。線種を選択する |
| 線が表示されない | レイヤー初期化のタイミング | 地図のロード完了を待つ（数秒待って再試行） |
| html-objects が空 | API 未起動またはエンドポイント不一致 | バックエンドの起動と `/api/html-objects` の応答を確認 |
| コンソールに CSS 警告が出る | MapLibre の非推奨疑似要素 | `npm install` で patch-package が適用される。patches/ フォルダを確認 |
