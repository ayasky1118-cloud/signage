# 本番環境の環境変数

API キーや DB 接続先など、環境に応じた設定を環境変数で行う方法。

---

## 方針

| 環境 | 設定方法 | 備考 |
|------|----------|------|
| **ローカル開発** | `backend/.env` に記述 | `.env.example` をコピーして `.env` を作成し、値を記入。`.env` は Git にコミットしない |
| **本番（Lambda / ECS 等）** | 実行環境の環境変数で指定 | `.env` ファイルはデプロイに含めず、AWS の環境変数などで渡す |

アプリは **環境変数を最優先** で読み、未設定時のみ `backend/.env`（ローカル用）を参照する。

---

## 必要な環境変数

### バックエンド（FastAPI）

| 変数名 | 必須 | 説明 | 例（ローカル） |
|--------|------|------|-----------------|
| `DATABASE_URL` | 本番では必須 | MySQL 接続 URL | `mysql+pymysql://root:password@127.0.0.1:3306/signage_dev?charset=utf8mb4` |
| `MAPTILER_API_KEY` | 任意 | MapTiler Geocoding（住所検証）用。未設定時は住所検証をスキップ | （MapTiler Cloud で発行したキー） |

### フロントエンド（Vue）

| 変数名 | 必須 | 説明 |
|--------|------|------|
| `VITE_API_BASE` | 任意 | バックエンドのベース URL。未設定時は Vite のプロキシ `/api` を使用 |

---

## ローカル開発の手順

1. `backend/.env.example` をコピーして `backend/.env` を作成する。
   ```bash
   cd backend
   cp .env.example .env
   ```
2. `.env` を開き、`DATABASE_URL` と必要に応じて `MAPTILER_API_KEY` を記入する。
3. `.env` は **コミットしない**（`.gitignore` で除外済み）。
4. **`.env` を変更したらバックエンドを再起動する**（設定は起動時にのみ読み込まれる）。

---

## 本番での設定例（AWS Lambda の場合）

Lambda 関数の「設定 → 環境変数」で以下を追加する想定。

| キー | 値（例） |
|------|----------|
| `DATABASE_URL` | `mysql+pymysql://user:pass@your-rds-host:3306/signage?charset=utf8mb4` |
| `MAPTILER_API_KEY` | （MapTiler で発行したキー） |

SAM テンプレートを使う場合は、`Globals.Function.Environment.Variables` または各関数の `Environment.Variables` に上記を定義する。  
**API キーやパスワードはテンプレートに直書きせず、AWS Systems Manager Parameter Store や Secrets Manager から参照する**ことを推奨する。
