# 本番環境の環境変数

API キーや DB 接続先など、環境に応じた設定を環境変数で行う方法。

---

## 方針

| 環境 | 設定方法 | 備考 |
|------|----------|------|
| **ローカル開発** | `backend/.env` に記述 | `.env.example` をコピーして `.env` を作成し、値を記入。`.env` は Git にコミットしない |
| **本番（Lambda / ECS 等）** | 実行環境の環境変数で指定 | `.env` ファイルはデプロイに含めず、AWS の環境変数などで渡す |

アプリは **環境変数を最優先** で読み、未設定時のみ `backend/.env`（ローカル用）を参照する。

---

## 必要な環境変数

### バックエンド（FastAPI）

| 変数名 | 必須 | 説明 | 例（ローカル） |
|--------|------|------|-----------------|
| `DATABASE_URL` | 本番では必須 | MySQL 接続 URL | `mysql+pymysql://root:password@127.0.0.1:3306/signage_dev?charset=utf8mb4` |
| `MAPTILER_API_KEY` | 任意 | MapTiler Geocoding（住所検証・ジオコーディング）用。未設定時は住所検証をスキップ、地図表示は 503 エラー | （MapTiler Cloud で発行したキー） |

### フロントエンド（Vue）

| 変数名 | 必須 | 説明 |
|--------|------|------|
| `VITE_API_BASE` | 任意 | バックエンドのベース URL。未設定時は Vite のプロキシ `/api` を使用 |
| `VITE_MAPTILER_API_KEY` | 任意 | MapTiler 地図タイル表示用。未設定時は地図が表示されず「VITE_MAPTILER_API_KEY を設定してください」と表示 |
| `VITE_COGNITO_USER_POOL_ID` | 認証利用時 | Cognito User Pool ID（main.ts で Amplify に渡す。未使用の場合は main.ts に直書き） |
| `VITE_COGNITO_USER_POOL_CLIENT_ID` | 認証利用時 | Cognito App Client ID |
| `VITE_LOGIN_COMPANY_ID` | 任意 | ログイン会社 ID。未設定時は 1 |

## ローカル開発の手順

### バックエンド

1. `backend/.env.example` をコピーして `backend/.env` を作成する。
   ```bash
   cd backend
   cp .env.example .env
   ```
2. `.env` を開き、`DATABASE_URL` と必要に応じて `MAPTILER_API_KEY` を記入する。
3. `.env` は **コミットしない**（`.gitignore` で除外済み）。
4. **`.env` を変更したらバックエンドを再起動する**（設定は起動時にのみ読み込まれる）。

### フロントエンド（地図表示を行う場合）

1. `frontend/.env.example` をコピーして `frontend/.env` を作成する。（Vite は `.env` を読み込む。`.env.example` を直接編集しても反映されない）
   ```bash
   cd frontend
   cp .env.example .env
   ```
2. `.env` を開き、`VITE_MAPTILER_API_KEY` に MapTiler Cloud で発行した API キーを記入する。
3. 未設定時は看板編集画面の地図表示エリアに「VITE_MAPTILER_API_KEY を設定してください」と表示される。
4. **`.env` を変更したらフロントエンド（Vite）を再起動する**（Vite の環境変数は起動時に読み込まれる）。

## 本番での設定例（AWS Lambda の場合）

Lambda 関数の「設定 → 環境変数」で以下を追加する想定。

| キー | 値（例） |
|------|----------|
| `DATABASE_URL` | `mysql+pymysql://user:pass@your-rds-host:3306/signage?charset=utf8mb4` |
| `MAPTILER_API_KEY` | （MapTiler で発行したキー） |

フロントエンドをビルドする際は、`VITE_MAPTILER_API_KEY` をビルド時の環境変数として渡す。  
（例: `VITE_MAPTILER_API_KEY=xxx npm run build`）

SAM テンプレートを使う場合は、`Globals.Function.Environment.Variables` または各関数の `Environment.Variables` に上記を定義する。  
**API キーやパスワードはテンプレートに直書きせず、AWS Systems Manager Parameter Store や Secrets Manager から参照する**ことを推奨する。
