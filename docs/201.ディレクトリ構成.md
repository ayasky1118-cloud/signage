# ディレクトリ構成

プロジェクト全体のフォルダ構成と、各フォルダの役割を説明する。

> **迷子用：** バックエンドだけを詳しく見たい場合は [202.backend_ディレクトリ構成.md](202.backend_ディレクトリ構成.md) を参照。

## 全体構成

```
signage/
├── frontend/       # フロントエンド（Vue 3 + Vite）
├── backend/        # バックエンド（FastAPI）
├── db/             # DBスキーマ・接続関連
├── docs/           # 手順書・仕様書
└── docker-compose.yml
```

### 主要ファイル

| ファイル | 説明 |
|----------|------|
| `pages/Menu.vue` | 注文・看板管理メニュー画面 |
| `pages/OrderList.vue` | 注文一覧・検索画面。メニューからの遷移時は従来どおり。order/main から遷移時（orderNo パラメータあり）は検索条件の注文番号に設定し初期検索を実施。戻るボタンは遷移元へ戻る。行のダブルクリックで注文詳細モーダル（OrderDetailModal）を表示する |
| `pages/OrderMain.vue` | 注文（新規・変更）画面。新規登録後、注文一覧へ遷移する場合は発番した注文番号をクエリパラメータ（orderNo）に設定して遷移する |
| `pages/OrderDetail.vue` | 看板編集画面。注文番号で検索・選択し、枝番タブで看板情報を編集する。「注文詳細」ボタンで注文詳細モーダル（OrderDetailModal）を表示。表示データは注文一覧API（searchOrders）の返却値を使用する |
| `layouts/DefaultLayout.vue` | 共通レイアウト（ヘッダー・ログアウトボタン・ページ領域） |
| `components/AppHeader.vue` | 共通ヘッダー（DefaultLayout から利用） |
| `components/OrderNoSelectModal.vue` | 注文番号選択モーダル（共通）。OrderMain・OrderList・OrderDetail の 3 画面で利用。一覧表示・選択時の挙動は各画面の `@select` で制御 |
| `components/OrderDetailModal.vue` | 注文詳細モーダル（共通）。注文一覧API（searchOrders）の返却値 OrderItem を基本情報・社内情報・備考・登録情報で表示。OrderList（行ダブルクリック）と OrderDetail（「注文詳細」ボタン）の 2 画面で利用 |
| `components/CustomerSelectModal.vue` | 顧客選択モーダル（共通）。OrderMain・OrderList で利用。選択時の反映は各画面の `@select` で制御（OrderList から起動時は担当者名を検索条件に反映しない） |
| `components/TemplateSelectModal.vue` | テンプレート選択モーダル（共通）。ログイン会社ID・顧客IDをパラメータに一覧を取得して表示。現状 OrderMain で利用 |
| `composables/useOrderApi.ts` | 注文一覧・注文1件取得（by-no）・注文新規登録 API 呼び出し |
| `assets/styles/common.css` | 共通スタイル（カラー変数、カードシャドウ等） |
| `assets/styles/order-list.css` | 注文一覧専用スタイル（テーブル、Flatpickr テーマ） |
| `assets/styles/order-detail.css` | 看板編集画面専用スタイル（カードヘッダー等） |

### 共通コンポーネント

複数画面で同じ UI を使う場合は `src/components/` に共通コンポーネントを置き、各ページから利用する。

- **OrderNoSelectModal.vue** … 注文番号選択モーダル。注文入力（OrderMain）、注文一覧の詳細検索（OrderList）、看板編集（OrderDetail）の 3 画面で「選択」ボタンから開く。表示内容・スタイルは共通で、選択時の処理だけ各画面の `@select` で実装する。
- **OrderDetailModal.vue** … 注文詳細モーダル。注文一覧API（GET /orders）の返却値 OrderItem を表示する。OrderList では行のダブルクリックで開き（一覧の行データをそのまま渡す）、OrderDetail では「注文詳細」ボタンで開く（検索済み注文番号で searchOrders を 1 件取得して渡す）。描画部分は共通。
- **CustomerSelectModal.vue** … 顧客選択モーダル。注文入力（OrderMain）と注文一覧（OrderList）で「顧客」の選択ボタンから開く。選択時は各画面の `@select` で反映する。OrderList から起動した場合は担当者名を検索条件のテキストに反映しない。
- **TemplateSelectModal.vue** … テンプレート選択モーダル。パラメータにログイン会社ID（必須）と顧客ID（任意）を取り、その条件でテンプレート一覧を API 取得して表示。現状は OrderMain で利用。他画面でも利用可能。

## backend/

FastAPI + SQLAlchemy によるバックエンド API。

| パス | 説明 |
|------|------|
| `app/` | アプリケーション本体 |
| `app/main.py` | エントリポイント。FastAPI インスタンス、CORS、ルーター登録 |
| `app/core/` | アプリ全体の設定 |
| `app/db/` | DB 接続・セッション管理 |
| `app/routers/` | API エンドポイント（ルーター） |
| `app/models/` | SQLAlchemy ORM モデル |
| `.env` | 環境変数（DATABASE_URL 等） |
| `check_db.py` | DB 接続確認用スクリプト |
| `requirements.txt` | Python 依存関係 |

### 主要ファイル

| ファイル | 説明 |
|----------|------|
| `app/main.py` | FastAPI アプリ、CORS、`/health`、ルーター登録 |
| `app/core/config.py` | 設定クラス（DATABASE_URL 等を .env から読み込み） |
| `app/db/session.py` | SQLAlchemy Engine、SessionLocal、get_db |
| `app/routers/address.py` | `GET /address/validate`（住所検証・MapTiler Geocoding） |
| `app/routers/company.py` | `GET /companies`（会社一覧、DB 疎通確認用） |
| `app/routers/customer.py` | `GET /customers`（顧客一覧。注文フォームの顧客選択用。company_id で絞り込み） |
| `app/routers/design_type.py` | `GET /design-types`（デザイン種別マスタ。company_id 指定時はその会社のみ、未指定時は全件） |
| `app/routers/order.py` | `GET /orders`（注文一覧検索。company_id 指定時は当該会社の注文のみ返す）、`GET /orders/by-no`（注文番号で1件取得・order_item・納期・校正予定日含む）、`POST /orders`（新規登録・customerId 指定・納期・校正予定日は任意） |
| `app/routers/template.py` | `GET /templates`（テンプレート一覧。注文フォームのテンプレート選択用。company_id で絞り込み） |
| `app/routers/template_item.py` | `GET /template-items`（テンプレート項目一覧。template_id で絞り込み） |
| `app/models/company.py` | Company テーブルの ORM モデル |
