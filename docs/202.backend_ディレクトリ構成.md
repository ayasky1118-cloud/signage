# バックエンド ディレクトリ構成

迷子になったとき用。バックエンドのフォルダ・ファイルの場所と役割。

---

## 構成図

```
signage/backend/
├── app/
│   ├── main.py              ← エントリポイント（FastAPI、CORS、ルーター登録）
│   ├── core/
│   │   └── config.py        ← 設定（DB接続URLなど）
│   ├── db/
│   │   └── session.py       ← DBセッション管理
│   ├── routers/
│   │   ├── address.py       ← GET /address/validate（住所検証）、GET /address/geocode（ジオコーディング・MapTiler）
│   │   ├── company.py       ← GET /companies（会社一覧）
│   │   ├── customer.py      ← GET /customers（顧客一覧・注文フォームの顧客選択用）
│   │   ├── design_type.py   ← GET /design-types（デザイン種別マスタ）
│   │   ├── html_object.py   ← GET /html-objects（HTMLオブジェクトマスタ・全社共通）
│   │   ├── order.py         ← GET /orders、GET /orders/by-no、PATCH /orders/by-no/details（枝番毎の備考・design_data）、POST /orders
│   │   ├── template.py      ← GET /templates（テンプレート一覧・注文フォームのテンプレート選択用）
│   │   └── template_item.py ← GET /template-items（テンプレート項目一覧）
│   └── models/
│       └── company.py       ← Company ORMモデル
├── check_db.py              ← DB接続確認用スクリプト
├── .env.example             ← 環境変数サンプル（コピーして .env を作成）
└── .env                     ← 環境変数（Git にコミットしない。本番では環境変数で設定）
```

---

## よく使う場所

| やりたいこと | 編集するファイル |
|--------------|------------------|
| 新しい API を追加する | `app/routers/` に新規作成 → `app/main.py` で登録 |
| API の URL や CORS を変える | `app/main.py` |
| DB 接続先を変える | `backend/.env` の DATABASE_URL（本番は環境変数） |
| API キー等の設定 | [203.本番環境の環境変数.md](203.本番環境の環境変数.md) 参照 |
| 注文関連 API（一覧検索・1件取得・登録）を変える | `app/routers/order.py` |

---

## 起動コマンド

```bash
cd signage/backend
source .venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

API は `http://localhost:8000` で動作。
