# バックエンド ディレクトリ構成

迷子になったとき用。バックエンドのフォルダ・ファイルの場所と役割。

---

## 構成図

```
signage/backend/
├── app/
│   ├── main.py              ← エントリポイント（FastAPI、CORS、ルーター登録）
│   ├── core/
│   │   └── config.py        ← 設定（DB接続URLなど）
│   ├── db/
│   │   └── session.py       ← DBセッション管理
│   ├── routers/
│   │   ├── company.py       ← GET /companies（会社一覧）
│   │   └── order.py         ← GET /orders（注文一覧検索）
│   └── models/
│       └── company.py       ← Company ORMモデル
├── check_db.py              ← DB接続確認用スクリプト
└── .env                     ← 環境変数（DATABASE_URL など）
```

---

## よく使う場所

| やりたいこと | 編集するファイル |
|--------------|------------------|
| 新しい API を追加する | `app/routers/` に新規作成 → `app/main.py` で登録 |
| API の URL や CORS を変える | `app/main.py` |
| DB 接続先を変える | `backend/.env` の DATABASE_URL |
| 注文一覧の検索ロジックを変える | `app/routers/order.py` |

---

## 起動コマンド

```bash
cd signage/backend
source .venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

API は `http://localhost:8000` で動作。
