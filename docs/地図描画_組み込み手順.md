# 地図描画 組み込み手順

sinage_old の地図描画実装を signage プロジェクトに組み込むための細かい手順です。

---

## 前提

- **元実装**: `/Users/yoshimi/dev/sinage_old/frontend/src/composables/` の useMap* 系
- **組み込み先**: `/Users/yoshimi/dev/signage/frontend/` の OrderDetail.vue + MapPreview.vue
- **データ形式**: signage の `design_data`（routes, texts, images, callouts）に合わせる

## Phase 2: 型定義の作成

### 手順 2-1: 地図データ型の定義ファイルを作成

`signage/frontend/src/types/map-data.ts` を作成する。

```typescript
// design_data と GeoJSON の橋渡し用型定義
import type { FeatureCollection, LineString, Point } from "geojson"

export interface RouteItem {
  id: string
  type: string  // value_code（例: ROAD_CLOSED, DETOUR）
  coordinates: [number, number][]
  width?: number
  color?: string
}

export interface MapDataJson {
  version: number
  branchNo?: string
  orderNo?: string
  routes?: RouteItem[] | FeatureCollection<LineString>
  texts?: FeatureCollection<Point>
  images?: FeatureCollection<Point>
  balloons?: FeatureCollection<Point>
  callouts?: FeatureCollection<Point>
}
```

※ signage の `design_data` は `routes`, `texts`, `images`, `callouts` を使用。sinage_old の `balloons` は `callouts` に相当。

### 手順 2-2: RouteItem と GeoJSON Feature の変換ユーティリティを追加

`map-data.ts` に以下を追加する。

- `routeItemToGeoJSONFeature(route: RouteItem): Feature<LineString>`
- `geoJSONFeatureToRouteItem(feature: Feature<LineString>): RouteItem`
- `routeItemsToFeatureCollection(routes: RouteItem[]): FeatureCollection<LineString>`

## Phase 4: useMapFeatures の作成

### 手順 4-1: useMapFeatures.ts をコピーして修正

1. `sinage_old/frontend/src/composables/useMapFeatures.ts` を `signage/frontend/src/composables/useMapFeatures.ts` にコピーする。

2. 以下を修正する。

   - **balloonFeatures**: プロパティ名を `callouts` に合わせるか、内部では `balloonFeatures` のままにして、外部との変換層で `callouts` にマッピングする。

3. `drawRoute` に `lineType`（value_code）や `lineStyle`（色・太さ）を渡せるようにする（Phase 6 で対応）。

## Phase 7: MapPreview の拡張

### 手順 7-1: MapPreview に map インスタンスの参照を公開

1. `MapPreview.vue` の `defineExpose` に `map` を追加する。
2. `map` が `null` でない場合に `ref` で保持し、`defineExpose({ fitMapToContainer, map })` のように公開する。

※ ただし、MapPreview は「表示専用」と「編集用」の2パターンがある。編集時は OrderDetail 側で map を直接触る必要があるため、**編集用は map を親から渡す方式**か、**MapPreview に `editMode` と `designData` を props で渡して内部でレイヤーを更新する方式**のどちらかを選ぶ。

**推奨**: MapPreview は「表示＋レイヤー更新」に限定し、クリック等のインタラクションは OrderDetail で map の `on('click')` を設定する。

### 手順 7-2: MapPreview に designData（routes 等）を props で受け取る

1. `MapPreview.vue` に props を追加する。

   ```typescript
   designData?: {
     routes?: RouteItem[] | FeatureCollection<LineString>
     texts?: FeatureCollection<Point>
     images?: FeatureCollection<Point>
     callouts?: FeatureCollection<Point>
   } | null
   ```

2. `watch` で `designData` の変更を監視し、map が存在する場合に `route`, `texts`, `images`, `callouts` 各ソースの `setData` を呼ぶ。

### 手順 7-3: map の load イベントでレイヤーを初期化

1. MapPreview の `initMap` 内で `map.on('load', ...)` を登録する。
2. コールバック内で `useMapLayers` の `initLayers` を呼ぶ。
3. `initLayers` に渡す features は、`designData` を GeoJSON 形式に変換したもの。初回は空の FeatureCollection でよい。

### 手順 7-4: map インスタンスを親に渡す

- `defineExpose({ fitMapToContainer, getMap })` のように `getMap` で map を返す。
- または `provide/inject` で OrderDetail から MapPreview の子孫に map を渡す。

## Phase 8: OrderDetail との統合

### 手順 8-1: OrderDetail で useMap* をインポート

OrderDetail.vue の script に以下を追加する。

```typescript
import { useMapFeatures } from "../composables/useMapFeatures"
import { useMapInteraction } from "../composables/useMapInteraction"
import { useMapHistory } from "../composables/useMapHistory"
import { useMapData } from "../composables/useMapData"
```

※ useMapLayers は MapPreview 内で使用するため、OrderDetail では不要。

### 手順 8-2: 全画面編集用の map コンテナを MapPreview に変更

現状、全画面編集では `MapPreview` を表示している。MapPreview が `initLayers` を呼ぶようにした場合、MapPreview 側で map の `load` イベントを待ってレイヤーを初期化する。

### 手順 8-3: 枝番ごとの mapDataByBranch と useMapFeatures の連携

- signage では `mapDataByBranch` が枝番ごとの `design_data` を保持している。
- useMapFeatures の `routeFeatures` 等は「現在編集中の枝番」のデータに対応させる必要がある。
- **方針**: 全画面編集を開いたとき、`activeBranch` の `mapDataByBranch[activeBranch]` を useMapFeatures の `restoreFeatures` に渡す。編集結果は `mapDataByBranch` に反映する。

### 手順 8-4: ルート描画のクリックハンドラを接続

1. MapPreview から `getMap()` で map インスタンスを取得する。
2. map の `load` 後に `useMapInteraction` の `setupClickHandler` を呼ぶ。
3. `onRoute` で `addTempMarker` を呼ぶ。
4. ルート描画ボタン押下で `drawRoute` を呼び、`mapDataByBranch` を更新する。

### 手順 8-5: design_data との同期

- `drawRoute` 実行後、`routeFeatures` を `RouteItem[]` に変換し、`mapDataByBranch.value[branchNo].routes` に設定する。
- 同様に、テキスト・画像・吹き出しの追加・削除時も `mapDataByBranch` を更新する。
- `designDataForApi` で API 送信時に `mapDataByBranch` から `routes` 等を取り出す。

## Phase 9: データ形式の統一

### 手順 9-1: design_data.routes の形式を決定

signage の `design_data` は `routes` を配列で持つ。各要素を次の形式にする。

```json
{
  "id": "uuid",
  "type": "ROAD_CLOSED",
  "coordinates": [[lng, lat], [lng, lat], ...]
}
```

GeoJSON の Feature 形式でも可。その場合は `geometry.type: "LineString"`, `geometry.coordinates` を使用。

### 手順 9-2: normalizeDesignDataToMapData の拡張

OrderDetail.vue の `normalizeDesignDataToMapData` で、`routes` が `RouteItem[]` 形式の場合に `FeatureCollection` に変換する処理を追加する。

## 注意事項

- MapPreview は「プレビュー用」と「全画面編集用」の2箇所で使われている。プレビュー用は `interactive=false` で表示のみ。編集用は `interactive=true` でクリック可能。レイヤー初期化は両方で行うが、クリックハンドラは編集用のときのみ登録する。
- 枝番切り替え時は、編集中の features を `mapDataByBranch` に保存してから、新しい枝番のデータを `restoreFeatures` で読み込む。
- sinage_old の `useMapInstance` は、signage では MapPreview が map を保持しているため、別 composable としては不要。MapPreview 内で initMap 相当の処理をしている。
