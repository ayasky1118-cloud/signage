# 地図表示と NAT ゲートウェイ

AWS 環境で、**住所ベースの地図中心表示**と**住所検証**に必要な NAT ゲートウェイの概要とコスト見積。

---

## 概要

NAT ゲートウェイが必要なのは、以下の 2 機能（いずれも Lambda が MapTiler API を呼び出すため）である。

| 機能 | 説明 |
|------|------|
| **住所ベースの地図中心表示** | 入力住所を座標に変換し、枝番エリア・全画面の地図の中心に表示する |
| **住所検証** | 入力住所の整合性チェック（注文フォーム登録時） |

※ 地図タイルの表示や線描画・アイコン追加などの編集操作はフロントエンド（ブラウザ）で完結するため、NAT ゲートウェイは不要。

### ジオコーディング失敗時の挙動

| エリア | 挙動 |
|--------|------|
| **枝番エリア** | エラーメッセージ表示、地図は表示されない |
| **全画面** | 地図は表示されるが、中心が東京駅のまま（入力住所が反映されない） |

Lambda は VPC 内で RDS Proxy に接続するため VPC に配置されているが、Lambda の ENI にはパブリック IP が付与されない。そのため、**インターネット上の MapTiler API に到達するには NAT ゲートウェイが必要**となる。

---

## 課金の仕組み

| 項目 | 内容 |
|------|------|
| **課金単位** | 稼働時間（1 時間あたり） |
| **料金目安** | 約 0.045 USD/時間（東京リージョン ap-northeast-1） |
| **1 日あたり** | 24 時間稼働で約 1.08 USD（約 160 円） |
| **1 ヶ月あたり** | 約 32 USD（約 4,800 円） |

※ 上記は NAT ゲートウェイの利用料のみ。データ転送料が別途発生する。

---

## 補足

- **起動中は常に課金**：NAT ゲートウェイは停止できないため、作成してから削除するまで料金が発生する
- **データ転送料**：NAT ゲートウェイ経由のデータ転送量にも料金がかかる（ジオコーディングの API 呼び出しは少量のため、月額数百円程度の範囲）
- **開発・検証用**：必要な期間のみ作成し、不要になったら削除することでコストを抑えられる

---

## コストを抑える方法

| 方法 | 説明 |
|------|------|
| **住所中心表示・住所検証を省略** | NAT なしで運用。地図編集（線描画・アイコン追加）は可能だが、地図の初期中心は東京駅のまま。住所検証はスキップされる |
| **フロントエンドで MapTiler を直接呼ぶ** | ジオコーディング・住所検証をバックエンド経由にせず、フロントエンドから MapTiler API を直接呼ぶ実装に変更すれば NAT 不要 |
| **期間限定で利用** | 検証・デモ時のみ NAT ゲートウェイを作成し、終了後に削除する |
| **本番運用** | 住所ベースの地図中心表示・住所検証が必要な場合は NAT ゲートウェイが必要。月額数千円程度のコストを見込む |

---

## 想定コストへの影響

| 構成 | 月額目安 |
|------|----------|
| **住所中心表示・住所検証なし**（NAT なし） | 約 4,000〜6,000 円 |
| **住所中心表示・住所検証あり**（NAT あり） | 約 9,000〜11,000 円 |

### 内訳（NAT ありの場合）

| リソース | 月額目安 |
|----------|----------|
| Lambda | 無料枠内〜数百円 |
| API Gateway | 無料枠内〜数百円 |
| RDS db.t3.micro | 約 2,000〜3,000 円 |
| RDS Proxy | 約 1,000〜2,000 円 |
| Secrets Manager | 約 400 円/シークレット |
| **NAT ゲートウェイ** | 約 4,000〜5,000 円 |
| **合計** | **約 9,000〜11,000 円** |

※ [02.Lambdaバックエンド構築.md](02.Lambdaバックエンド構築.md) の「想定コスト」をベースに、NAT ゲートウェイ分を加算。
