# 00. 環境準備手順（別 PC 用）

別の PC（Mac / Windows）で AWS デプロイ関連の作業を行うために必要な環境構築手順。  
この手順に従えば、新しいマシンでも同じ環境を再現できる。

---

## 必要なツール一覧

| ツール | 用途 | 必須 |
|--------|------|------|
| AWS CLI | Lambda・RDS 等の操作 | ◎ |
| Docker | イメージビルド・プッシュ、ローカル動作確認 | ◎ |
| Git | リポジトリの取得 | ◎ |
| Node.js | フロントエンド開発（ローカル時） | △ |
| Python | バックエンド開発（ローカル時） | △ |

※ Lambda デプロイのみ行う場合は Docker と AWS CLI があれば可。ローカル開発も行う場合は Node.js・Python も必要。

---

## 1. AWS CLI のインストール

### 1.1 macOS

**方法 A: Homebrew（推奨）**
```bash
brew install awscli
```

**方法 B: 公式インストーラー**
1. [AWS CLI v2 インストーラー（macOS）](https://awscli.amazonaws.com/AWSCLIV2.pkg) をダウンロード
2. `AWSCLIV2.pkg` をダブルクリックしてインストール
3. ターミナルを再起動

### 1.2 Windows

**方法 A: MSI インストーラー（推奨）**
1. [AWS CLI v2 インストーラー（Windows）](https://awscli.amazonaws.com/AWSCLIV2.msi) をダウンロード
2. ダウンロードした `.msi` を実行
3. インストール後、**コマンドプロンプトまたは PowerShell を再起動**

**方法 B: winget**
```powershell
winget install Amazon.AWSCLI
```

### 1.3 インストール確認

```bash
aws --version
```

例: `aws-cli/2.x.x ...` と表示されれば OK。

## 2. AWS CLI の設定

### 2.1 アクセスキーの用意

1. [AWS マネジメントコンソール](https://console.aws.amazon.com/) にログイン
2. 右上のアカウント名 → **セキュリティ認証情報**
3. **アクセスキー** → **アクセスキーを作成**
4. **アクセスキー ID** と **シークレットアクセスキー** を控える（シークレットは作成時のみ表示）

### 2.2 設定の実行

```bash
aws configure
```

| プロンプト | 入力内容 |
|------------|----------|
| AWS Access Key ID | 控えたアクセスキー ID |
| AWS Secret Access Key | 控えたシークレットアクセスキー |
| Default region name | `ap-northeast-1` |
| Default output format | `json`（そのまま Enter でも可） |

### 2.3 設定確認

```bash
aws configure list
aws sts get-caller-identity
```

- `access_key` と `secret_key` が表示されていれば設定済み
- `get-caller-identity` でアカウント情報が返れば認証 OK

## 3. Docker のインストール

### 3.1 macOS

1. [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/) をダウンロード
2. インストール後、Docker Desktop を起動
3. メニューバーにクジラアイコンが表示されれば起動済み

### 3.2 Windows

1. [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/) をダウンロード
2. インストール（WSL 2 の有効化を求められた場合は従う）
3. インストール後、Docker Desktop を起動

### 3.3 インストール確認

```bash
docker --version
docker compose version
```

---

## 4. Git のインストール

### 4.1 macOS

通常は Xcode Command Line Tools に含まれる。未インストールの場合:

```bash
xcode-select --install
```

または [Git 公式](https://git-scm.com/download/mac) からインストール。

### 4.2 Windows

1. [Git for Windows](https://git-scm.com/download/win) をダウンロード
2. インストーラーの指示に従ってインストール
3. インストール後、コマンドプロンプトまたは PowerShell を再起動

### 4.3 インストール確認

```bash
git --version
```

## 5. プロジェクトの取得

```bash
# リポジトリをクローン（例）
git clone <リポジトリURL> signage
cd signage
```

---

## 6. 動作確認（Lambda デプロイ準備まで）

以下が通れば、Lambda デプロイに必要な環境は整っている。

```bash
# AWS 認証
aws sts get-caller-identity

# Docker ビルド（プロジェクトルートで）
cd signage
docker compose build backend
docker compose up -d db backend

# ヘルスチェック
curl http://localhost:8000/health
# → {"status":"ok"}
```

---

## 7. トラブルシューティング

| 現象 | 対処 |
|------|------|
| `aws: command not found` | ターミナルを再起動。パスが通っていない場合はインストールパスを確認 |
| `aws configure list` で全て `<not set>` | `aws configure` を再実行。認証情報ファイル `~/.aws/credentials` の存在を確認 |
| `docker: command not found` | Docker Desktop が起動しているか確認。ターミナルを再起動 |
| Windows で `docker compose` が動かない | Docker Desktop のバージョンを確認（v2 以降）。`docker-compose`（ハイフン）でも可 |
