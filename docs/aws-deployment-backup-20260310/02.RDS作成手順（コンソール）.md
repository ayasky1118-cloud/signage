# 02. RDS for MySQL 作成手順（コンソール）

Phase 2 の RDS 作成を AWS コンソールで行う際の詳細手順。やり取りで判明した点を含む。

---

## 前提

- Phase 1（VPC）完了。デフォルト VPC を使用する場合は確認のみ。
- デフォルト VPC の確認: `aws ec2 describe-vpcs --filters "Name=is-default,Values=true" --region ap-northeast-1`

---

## 1. RDS を開く

1. AWS マネジメントコンソールにログイン
2. 検索バーで「**RDS**」と入力 → **RDS** を選択
3. リージョンが **東京（ap-northeast-1）** であることを確認

---

## 2. データベースの作成を開始

1. 左メニュー **「データベース」** をクリック
2. **オレンジ色の「データベースを作成する」** ボタンをクリック

---

## 3. エンジンとテンプレート

| 項目 | 選択 |
|------|------|
| エンジンのタイプ | **MySQL** |
| エディション | **MySQL Community** |
| エンジンバージョン | **MySQL 8.0.x**（デフォルト） |
| テンプレート | **開発/テスト** |

---

## 4. デプロイオプション

**「シングル AZ DB インスタンスデプロイ (1 インスタンス)」** を選択。

| オプション | 用途 |
|------------|------|
| マルチ AZ DB クラスター (3 インスタンス) | 本番・高可用性 |
| マルチ AZ DB インスタンス (2 インスタンス) | 本番・フェイルオーバー |
| **シングル AZ DB インスタンス (1 インスタンス)** | **開発・テスト（推奨）** |

開発用は 1 インスタンスで十分。コストも抑えられる。

---

## 5. 設定

| 項目 | 入力 |
|------|------|
| DB インスタンス識別子 | `signage-dev-db` |
| マスターユーザー名 | `admin` |
| マスターパスワード | 任意（必ず控える） |
| パスワードの確認 | 上と同じ |

---

## 6. インスタンス構成

| 項目 | 選択 |
|------|------|
| DB インスタンスクラス | **バースト可能クラス** → **db.t3.micro** |

---

## 7. ストレージ

| 項目 | 選択 |
|------|------|
| ストレージタイプ | **汎用 SSD (gp3)** |
| 割り当てストレージ | **20** GiB |
| ストレージの自動スケーリング | オフで可 |

開発用なら 20 GiB で十分。

---

## 8. 接続

| 項目 | 選択・入力 |
|------|------------|
| コンピューティングリソース | **接続先を指定しない** |
| VPC | **デフォルト VPC** |
| DB サブネットグループ | **default** |
| パブリックアクセス | **あり**（ローカルからスキーマ投入するため） |
| VPC セキュリティグループ | **新規作成** |
| 新規セキュリティグループ名 | `signage-db-sg` |
| アベイラビリティゾーン | **なし（推奨）** |

※ 本番ではパブリックアクセスを「いいえ」にすることを推奨。

---

## 9. 認証情報管理

**「セルフマネージド」** を選択。

| 選択肢 | 内容 |
|--------|------|
| **セルフマネージド** | 自分でパスワードを設定。Phase 3 で Secrets Manager に手動登録する |
| AWS Secrets Manager で管理 | RDS が自動でシークレット作成。追加料金・一部制限あり |

**暗号化キーを選択**: 「aws/secretsmanager（デフォルト）」のままで可。

---

## 10. データベース認証オプション

**「パスワード認証」** を選択。

- **場所**: 「認証情報管理」セクション内の **「その他の認証情報の設定」** を展開 → **「データベース認証オプション」**
- 見つからない場合: デフォルトがパスワード認証のことが多いので、そのままで可

---

## 11. RDS Proxy

1. RDS 作成画面を下にスクロールし、**「RDS Proxy」** または **「プロキシ」** の項目を探す
2. **チェック OFF** のままにする（デフォルトで OFF のことが多い）
3. RDS Proxy は Phase 4 で RDS 作成後に別途作成する

---

## 12. 追加設定

1. **「追加設定」** セクションを展開する
2. 以下の項目を確認・設定する:

| 項目 | 設定 |
|------|------|
| データベースポート | **3306**（デフォルトのまま） |
| データベース名 | `signage_dev`（設定画面にあれば入力） |

※ 「追加設定」にデータベース名の項目がない場合:
- 「設定」セクションや「追加設定」を下までスクロールして確認
- 見つからなければ作成後に接続して `CREATE DATABASE signage_dev;` で作成しても可

---

## 13. 概算コストの確認

画面下部の「概算月間コスト」を確認する。

| 項目 | 目安 |
|------|------|
| DB インスタンス | 約 19 USD/月 |
| ストレージ | 約 3 USD/月 |
| **合計** | **約 22 USD/月（日本円で約 3,300 円/月）** |

※ RDS Proxy、Lambda、API Gateway 等は別料金。

---

## 14. 作成の実行

1. 画面を下までスクロールし、**「データベースの作成」** ボタンをクリック
2. 作成が開始される。RDS の「データベース」一覧に新しい DB が表示される
3. ステータスが「**利用可能**」になるまで待つ（おおよそ 5〜10 分）
4. ステータスが「利用可能」になったら次へ進む

---

## 15. 作成後の確認とエンドポイントの控え

1. **RDS** → **データベース** を開く
2. 作成した `signage-dev-db` をクリック
3. **「接続」** または **「を使用して接続」** をクリック
4. **エンドポイント** を控える:
   - 「接続とセキュリティ」タブに「エンドポイント」欄があればそこから取得
   - ない場合は **「接続」** 画面の接続コマンド内、**`-h` の直後** がエンドポイント  
     例: `mysql -h signage-dev-db.xxxxx.ap-northeast-1.rds.amazonaws.com ...` の `signage-dev-db.xxxxx.ap-northeast-1.rds.amazonaws.com`
5. **ポート**: `3306`（コマンドの `-P` の直後）

---

## 16. セキュリティグループ（3306 を許可）

RDS に接続するには、先にセキュリティグループで 3306 を許可する必要がある。

1. **EC2** コンソールを開く（検索バーで「EC2」と入力）
2. 左メニュー **「セキュリティグループ」** をクリック
3. `signage-db-sg` を検索して選択
4. **「インバウンドルール」** タブで **「インバウンドルールを編集」** をクリック
5. **「ルールを追加」** をクリック
6. 以下を設定:
   - **タイプ**: MySQL/Aurora
   - **ポート**: 3306（自動入力）
   - **ソース**: **マイ IP**（自分の PC からのみ許可）または **0.0.0.0/0**（開発用・全許可。本番では非推奨）
7. **「ルールを保存」** をクリック

これでローカル PC から RDS に接続できるようになる。

---

## 17. 接続手順（スキーマ投入・接続確認用）

### 接続コマンドの取得方法

1. **RDS** → **データベース** → `signage-dev-db` をクリック
2. **「接続」** または **「を使用して接続」** をクリック
3. 以下の項目を設定（**デフォルトのままでよい**）:

| 項目 | 選択 | 備考 |
|------|------|------|
| **接続方法** | **コードスニペット** | ターミナルで `mysql` コマンドを実行する場合。デフォルトで選択済み |
| （未選択） | クラウドシェル | AWS コンソール内のシェルから接続する場合。今回は不要 |
| （未選択） | エンドポイント | IDE（DBeaver 等）から接続する場合。ホスト名だけ欲しいとき |
| **プログラミング言語** | **MySQL (macOS)** | Mac の場合。Windows の場合は「MySQL (Windows)」を選択 |
| **エンドポイントタイプ** | **インスタンスエンドポイント** | デフォルトのまま。通常はインスタンスに直接接続 |

※ 「インターネットアクセスゲートウェイ」「IAM 認証」は、画面上部や接続方法の近くに小さく表示されることがある（例: 丸にマイナスで「無効」）。**表示されていなくても問題ない**。設定変更は不要で、パスワード認証で接続する。

4. **接続手順** に表示される `mysql` コマンドをコピー

### 接続コマンドの例

`<パスワード>` を実際のパスワードに置換してコピペ実行:

```bash
mysql -h signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com -P 3306 -u admin -p'<パスワード>' --ssl-mode=REQUIRED signage_dev
```

※ MySQL 8.0 の `caching_sha2_password` は SSL 接続を要求するため、`--ssl-mode=REQUIRED` が必要。`--ssl-verify-server-cert` は非対応のクライアントがある。

### 接続確認

```bash
mysql -h signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com -P 3306 -u admin -p'<パスワード>' --ssl-mode=REQUIRED signage_dev -e "SELECT 1"
```

成功すれば `1` が表示される。
