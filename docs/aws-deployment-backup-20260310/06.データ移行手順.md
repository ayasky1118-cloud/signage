# データ移行手順（ローカル ↔ AWS RDS）

ローカル Docker の MySQL と AWS RDS の間でデータを移行・復元する手順。

---

## 前提

- **ローカル**: Docker で MySQL が起動している（`docker compose up -d`）
- **ローカル**: `mysql` および `mysqldump` クライアントがインストール済み（`brew install mysql-client`）
- **RDS**: パブリックアクセス「**あり**」で作成済み
- **RDS**: セキュリティグループでマイ IP から 3306 を許可済み

---

## RDS インスタンスと RDS Proxy の違い

| エンドポイント | 用途 | ローカルから接続 |
|----------------|------|------------------|
| **RDS インスタンス**（`signage-dev-db.xxx.rds.amazonaws.com`） | スキーマ投入・データ移行・復元 | ◎ 可能 |
| **RDS Proxy**（`signage-dev-proxy.proxy-xxx.rds.amazonaws.com`） | Lambda からの接続用 | × 不可（VPC 内のみ） |

**データ移行・復元では RDS インスタンスのエンドポイントを使用する。**

---

## ローカル → RDS（データ移行）

### 1. 事前: RDS にスキーマを投入（初回のみ）

RDS が空の場合は、ローカルでスキーマを適用した DB から移行すればスキーマも含まれる。または RDS に直接 DDL を流す:

```bash
# ローカルで DDL を RDS に流す例（RDS_HOST, RDS_USER, RDS_PASSWORD を事前に export）
cd /Users/yoshimi/dev/signage
export RDS_HOST="signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com"
export RDS_USER="admin" RDS_PASSWORD="<パスワード>"
for f in db/00_schema/001_company.sql db/00_schema/002_user.sql db/00_schema/003_customer.sql db/00_schema/004_template.sql db/00_schema/005_template_item.sql db/00_schema/006_order_no_seq.sql db/00_schema/007_design_type.sql db/00_schema/008_order_main.sql db/00_schema/009_order_item.sql db/00_schema/010_order_detail.sql db/00_schema/011_html_object.sql db/00_schema/012_html_object_value.sql; do
  mysql -h"$RDS_HOST" -u"$RDS_USER" -p"$RDS_PASSWORD" --ssl-mode=REQUIRED signage_dev < "$f"
done
```

※ ローカル DB が最新スキーマ（011, 012 含む）であれば、`migrate-to-rds.sh` のダンプにスキーマも含まれる。

### 2. 移行を実行

`<パスワード>` を Secrets Manager の値に置換:

```bash
cd /Users/yoshimi/dev/signage
RDS_HOST="signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com" \
RDS_USER="admin" RDS_PASSWORD="<パスワード>" \
./scripts/migrate-to-rds.sh signage_dev
```

※ `migrate-to-rds.sh` はローカル → RDS の一方向。**ローカル DB には書き込まない。**  
※ RDS 側の既存テーブルは**上書き**される。RDS に重要なデータがある場合は事前にバックアップすること。

### 3. 動作確認

1. https://main.d27nzen1vhtcee.amplifyapp.com にアクセス
2. Cognito でログイン
3. 注文一覧・メニュー等でデータが表示されることを確認

---

## RDS → ローカル（復元）

ローカル DB が壊れた場合、RDS から復元する。移行後に RDS には最新データが入っているため、それをローカルに戻す。

```bash
cd /Users/yoshimi/dev/signage
RDS_HOST="signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com" \
RDS_USER="admin" RDS_PASSWORD="<パスワード>" \
./scripts/migrate-from-rds.sh signage_dev
```

※ ローカルの既存データは**上書き**される。確認プロンプトが表示される。  
※ RDS の mysqldump には `--set-gtid-purged=OFF` を指定（RDS の権限制限に対応）。

### 復元後に 011, 012 テーブルがない場合

RDS に 011, 012 が含まれていなかった場合、ローカルで DDL を適用する:

```bash
cd /Users/yoshimi/dev/signage
docker exec -i signage-db mysql -uroot -ppassword --default-character-set=utf8mb4 signage_dev < db/00_schema/011_html_object.sql
docker exec -i signage-db mysql -uroot -ppassword --default-character-set=utf8mb4 signage_dev < db/00_schema/012_html_object_value.sql
docker exec -i signage-db mysql -uroot -ppassword --default-character-set=utf8mb4 signage_dev < db/01_seed/010_html_object_seed.sql
```

---

## 注意

- **user テーブルの auth_uid**: Cognito の **sub** と一致させる必要がある。下記「auth_uid の設定」を参照。
- **接続エラー**: RDS のセキュリティグループで 3306 がマイ IP から許可されているか確認する。

---

## RDS がパブリックアクセス「いいえ」の場合

ローカルから接続できない場合は、以下のいずれかでデータ移行を行う。

1. **AWS CloudShell** から実行（CloudShell は VPC 内のリソースにアクセス可能）
2. **EC2 bastion** を立て、SSH 経由で `mysql` を実行
3. **RDS のパブリックアクセスを一時的に「あり」に変更**（開発環境のみ。移行後に「いいえ」に戻す）

---

## auth_uid の設定（ログイン後に「ログイン確認中...」で戻る場合）

ログイン成功後、AuthGuard が `user` マスタの `auth_uid` と Cognito の **sub** を突合する。一致しないとログイン画面に戻る。

### 手順

1. **Cognito の sub を確認**
   - AWS コンソール → Cognito → ユーザープール `ap-northeast-1_YCSDgFpIR` → ユーザー
   - ログインに使うユーザーを選択 → **サブ（sub）** をコピー（例: `a1b2c3d4-e5f6-7890-abcd-ef1234567890`）

2. **user テーブルの auth_uid を更新**
   ```sql
   -- RDS に接続して実行
   UPDATE user SET auth_uid = 'Cognitoのsubをここに貼り付け' WHERE user_id = 1;
   ```

3. **確認**
   ```sql
   SELECT user_id, email, auth_uid FROM user;
   ```

4. Amplify で再度ログインしてメニュー画面が表示されるか確認する。

## ローカル DB の復元（RDS → ローカル）

ローカル DB が壊れた場合、RDS から復元できる。移行後に RDS には最新データが入っているため、それをローカルに戻す。

```bash
cd /Users/yoshimi/dev/signage
RDS_HOST="signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com" \
RDS_USER="admin" RDS_PASSWORD="<パスワード>" \
./scripts/migrate-from-rds.sh signage_dev
```

※ ローカルの既存データは上書きされる。確認プロンプトが表示される。
