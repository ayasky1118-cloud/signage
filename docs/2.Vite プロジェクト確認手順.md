# Vite プロジェクト確認手順（Vue3 + TypeScript）

本ドキュメントは、現在のディレクトリが正しく Vite プロジェクトであるかを確認するための手順である。
Tailwind や Router を導入する前に実施する。

---

## 1. 作業ディレクトリへ移動

Vite フロントエンドのルートへ移動する。

```bash
cd /path/to/frontend
```

---

## 2. 必須ファイル確認

```bash
ls
```

以下のファイル／ディレクトリが存在すること。

```
package.json
vite.config.ts（または vite.config.js）
index.html
src/
```

---

## 3. package.json の確認

```bash
cat package.json
```

以下が含まれていること。

- `"vite"` が dependencies または devDependencies に存在
- `"scripts"` に `"dev"` が存在

例：

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}
```

---

## 4. src 構造確認

```bash
ls src
```

最低限以下が存在すること。

```
main.ts
App.vue
```

---

## 5. 開発サーバー起動確認

```bash
npm run dev
```

ターミナルに以下のような表示が出ること。

```
VITE vX.X.X  ready in XXX ms
➜  Local:   http://localhost:5173/
```

**ポートについて：**  
5173 が既に使用中の場合、Vite は自動的に次の空きポート（例：5174）を使用する。  
表示された URL（例：`http://localhost:5174/`）でアクセスすること。

ブラウザでアクセスし、画面が表示されれば Vite プロジェクトは正常。

---

## 6. 異常ケースチェック

### ❌ npm run dev が動かない

→ node_modules 未インストールの可能性

```bash
npm install
```

---

### ❌ vite command not found

→ package.json に vite が無い
→ 依存が壊れている

```bash
npm install vite -D
```

---

## 判定

以下が全て OK であれば、Vite プロジェクトとして正常。

- package.json に vite がある
- main.ts / App.vue がある
- npm run dev が起動する
- 表示された localhost の URL（5173 または 5174 等）で画面が表示される

---
