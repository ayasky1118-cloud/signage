# GitHub vs GitLab 比較検討表

> **重要要件**: 「設定ほぼゼロで自動パイプライン」を実現したい場合の比較

---

## 1. 料金プラン比較

| 項目 | GitHub | GitLab |
|------|--------|--------|
| **Free プラン** | $0/ユーザー/月 | $0/ユーザー/月 |
| **最低有料プラン** | Team: $4/ユーザー/月 | Premium: $29/ユーザー/月 |
| **年間費用（Free）** | 0円 | 0円 |
| **年間費用（有料・1人）** | 約7,200円 | 約52,200円 |

---

## 2. Auto DevOps / ゼロコンフィグパイプライン（最重要）

| 項目 | GitHub | GitLab |
|------|--------|--------|
| **自動パイプライン生成** | × なし | ◎ **Auto DevOps 搭載** |
| **設定の手間** | ワークフローを手動で作成 | **有効化するだけで自動動作** |
| **言語・フレームワーク検出** | △ テンプレート提案のみ | ◎ **自動検出してビルド** |
| **自動で行われること** | なし（全て手動設定） | ビルド・テスト・セキュリティスキャン・デプロイ・監視 |
| **結論** | 手動設定が必須 | **要件を満たす** |

---

## 3. CI/CD 比較

| 項目 | GitHub | GitLab |
|------|--------|--------|
| **CI/CD ツール** | GitHub Actions | GitLab CI/CD |
| **Free プラン分数** | 2,000分/月 | 400分/月 |
| **Self-hosted Runner** | 分数無制限・無料 | 分数無制限・無料 |
| **設定ファイル** | `.github/workflows/*.yml`（手動作成） | `.gitlab-ci.yml`（Auto DevOps なら不要） |

※ 1人運用で 400分を超える場合は Self-hosted Runner で無制限にできる

---

## 4. 主な機能比較

| 機能 | GitHub Free | GitLab Free |
|------|-------------|-------------|
| **リポジトリ** | 無制限（public/private） | 無制限 |
| **Issues / Projects** | ○ | ○ |
| **Pull Request / Merge Request** | ○ | ○ |
| **パッケージストレージ** | 500MB | **10 GiB** |
| **Container Registry** | △（有料で拡張） | ◎ **10 GiB 標準** |
| **Code Quality レポート** | △（Actions で手動実装） | ◎ **標準搭載** |
| **Container Scanning** | △（Actions で手動実装） | ◎ **標準搭載** |
| **Dependabot / 依存関係スキャン** | ○ | ○ |

---

## 5. 1人運用での使いやすさ

| 観点 | GitHub | GitLab |
|------|--------|--------|
| **初期セットアップの手間** | △ ワークフロー作成が必要 | ◎ Auto DevOps で即開始 |
| **学習コスト** | ◎ 低い（ただし設定は自分で） | ○ やや高い（ただし自動化で軽減） |
| **日本語情報量** | ◎ 多い | ○ 少なめ |
| **UI のシンプルさ** | ◎ シンプル | △ 多機能 |
| **Vue.js との相性** | ◎ 例が多い | ○ 問題なし（Auto DevOps で自動検出） |

---

## 6. 制限・注意点

| 項目 | GitHub Free | GitLab Free |
|------|-------------|-------------|
| **ユーザー数制限** | なし | グループあたり5人まで（1人なら問題なし） |
| **CI/CD 分数超過時** | 従量課金 | 従量課金（$10/1,000分）または Self-hosted Runner |
| **Auto DevOps 利用条件** | ー | Kubernetes クラスタ + Dockerfile または Buildpack |

---

## 7. 総合評価（1人・signage プロジェクト・Auto DevOps 希望）

| 評価項目 | GitHub | GitLab |
|----------|--------|--------|
| **ゼロコンフィグパイプライン** | × 不可 | ◎ **Auto DevOps で実現** |
| **コスト** | ◎ 0円 | ◎ 0円 |
| **CI/CD 分数** | ◎ 2,000分/月 | △ 400分/月（Self-hosted で解消可） |
| **DevOps 統合度** | △ 個別に設定 | ◎ **一括で自動** |
| **運用の手間（初期）** | △ 設定作業あり | ◎ **最小限** |
| **推奨度** | ○ 手動設定OKなら | ◎ **推奨** |

---

## 8. 結論・選び方

### 総合結論

**「設定ほぼゼロで自動パイプライン」を重視するなら → GitLab を推奨**

GitHub には Auto DevOps 相当の機能がなく、パイプラインは必ず手動で設定する必要があります。GitLab の Auto DevOps なら、有効化するだけでビルド・テスト・セキュリティスキャン・デプロイまで自動化されます。

| こんな場合 | おすすめ |
|------------|----------|
| **設定ほぼゼロで自動パイプラインが欲しい** | **GitLab** ◎ |
| CI/CD の設定を自分で細かく作り込みたい | GitHub |
| CI/CD 分数を多く使いたい（且つ Self-hosted 不可） | GitHub |
| コンテナ・セキュリティを最初から統合したい | **GitLab** ◎ |
| 日本語の情報量を最優先したい | GitHub |
| 1人で手早く DevOps を始めたい | **GitLab** ◎ |

---

### signage プロジェクト向けの推奨

- **要件**: Auto DevOps 的なゼロコンフィグ、1人運用、無料
- **推奨**: **GitLab Free**
- **理由**: Auto DevOps で即パイプライン構築、400分は Self-hosted Runner で補える、Container Registry 10 GiB も無料
