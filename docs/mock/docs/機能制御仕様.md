# 各機能の制御仕様

本ドキュメントは、注文・看板管理アプリの各機能における制御部分（イベントハンドラ、状態管理、ロジック）を整理したものです。

---

## 1. メニュー（menu.html）

### 概要
- 静的リンクによる画面遷移のみ
- 制御用のJavaScriptはなし

### 操作
| 操作 | 遷移先 |
|------|--------|
| 注文情報入力 / 変更 | `order_main.html` |
| 看板情報入力 / 変更 | `order_detail.html` |
| 注文 / 看板情報一覧 | `order_list.html` |

---

## 2. 注文情報入力 / 変更（order_main.html）

### 2.1 モード切替

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `radioModeNew` / `radioModeChange` | `change` | 新規/変更モードの切り替え。変更→新規で未保存変更がある場合、`unsavedChangesConfirmModal` を表示。新規→変更で入力変更がある場合、`changeNoticeModal` を表示。 |

### 2.2 モード別制御

| 状態 | 制御内容 |
|------|----------|
| 新規モード | 注文番号は自動採番、会社選択・テンプレート選択が有効。登録ボタンは「登録」。 |
| 変更モード（一覧から遷移） | 注文番号は読み取り専用。検索/選択ボタンは無効。 |
| 変更モード（メニューから遷移） | 注文番号は編集可能。検索/選択ボタンで注文データを取得後、他項目が編集可能になる。 |

### 2.3 注文番号関連

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `btnOrderNoList` | `click` | 注文番号選択モーダル（`orderNoSelectModal`）を開く |
| `btnOrderNoSearch` | `click` | 必須チェック後、`setDummyOrderData()` でダミーデータを反映。`hasSearchedInChangeMode = true` に更新。 |
| `inputOrderNo` | `input` / `change` | 変更モード時、注文番号変更で他項目をクリアし `clearOtherFieldsOnOrderNoChange()` を実行 |
| `orderNoSelectItem`（行クリック） | `click` | 選択した注文番号を入力欄に反映し、他項目をクリア |

### 2.4 会社選択

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `btnCompanyList` | `click` | 会社選択モーダル（`companySelectModal`）を開く |
| `companySelectItem` | `click` | 行を選択状態（`selectedCompanyIndex`）に更新 |
| `companySelectModalOk` | `click` | 選択した会社を反映し、テンプレートを初期化 |
| `companySelectModalClear` | `click` | 会社名・担当者をクリア |
| `companySelectModalCancel` / Backdrop | `click` | モーダルを閉じる |

### 2.5 テンプレート

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `templateSelect` | `change` | `renderTemplateItems()` でテンプレート項目を動的生成・表示 |
| `btnTemplateSelect` | `click` | テンプレート選択（UI による選択） |

### 2.6 登録・バリデーション

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `btnRegister` | `click` | `validateRequiredFields()` で必須チェック。NG なら `requiredValidationModal`。OK なら `registerConfirmModal` を表示 |
| `registerConfirmModalOk` | `click` | 登録確認後、`registerResultModal` を表示（TODO: API 呼び出し） |
| `registerConfirmModalCancel` / Backdrop | `click` | 登録確認モーダルを閉じる |

### 2.7 未保存変更確認

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `linkBack` | `click` | `hasUnsavedChanges()` で未保存変更があれば `unsavedChangesConfirmModal` を表示。なければ `history.back()` |
| `linkHeaderMenu` | `click` | 未保存変更があれば確認モーダル表示後、`menu.html` へ遷移 |
| `linkOrderDetail` | `click` | 未保存変更があれば確認モーダル表示後、`order_detail.html` へ遷移 |
| `unsavedChangesConfirmModalOk` | `click` | 破棄して `executePendingUnsavedAction()` を実行（新規切替・戻る・他画面へ等） |

### 2.8 変更確認モーダル（新規→変更で入力変更あり）

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `changeNoticeModalDiscard` | `click` | フォームをクリアし、変更モードへ切り替え |
| `changeNoticeModalRegister` | `click` | モーダルを閉じ、登録確認モーダルを開く |
| `changeNoticeModalCancel` | `click` | モーダルを閉じ、新規モードのまま維持 |

### 2.9 初期化・URL パラメータ

| 処理 | 制御内容 |
|------|----------|
| `DOMContentLoaded` | URL の `mode=edit` で変更モードを強制。`orderNo` 等のパラメータがあればフォームに反映 |
| `AppConfig.onPageshowReplaceWithMainUrl` | bfcache 復元時に `pageshow` で URL をメインURL に置換 |

## 3. 看板情報入力 / 変更（order_detail.html）

### 3.1 ページモード

| 状態 | 判定 | 制御内容 |
|------|------|----------|
| 登録モード | `mode=edit` でない | メニューからの新規。検索ボタンで注文取得後に表示 |
| 変更モード | `mode=edit` | 一覧からの遷移。URL パラメータで注文情報を表示 |

### 3.2 注文情報表示の開閉

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `orderInfoToggle` | `click` | `orderInfoBody` の表示/非表示を切り替え。テキスト「表示」「閉じる」を更新 |

### 3.3 注文番号・検索

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `btnOrderSelect` | `click` | 注文番号選択モーダル（`orderNoSelectModal`）を開く |
| `btnOrderSearch` | `click` | 注文番号未入力なら `orderSearchValidationModal`。入力済みなら `setDummyOrderData()` でダミーデータを反映 |
| `inputOrderNo` | `input` / `change` | 変更時に `checkActuallyDirty()` で未保存変更があれば `pendingAction` で注文番号変更を保留 |
| `orderNoSelectItem` | `click` | 選択した注文番号を反映。`applyBranchTabsForOrder()` で枝番タブを更新 |

### 3.4 枝番タブ

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `branchTab`（タブクリック） | `click` | `setActiveBranchTab()` で枝番切り替え。未保存変更があれば `branchSwitchConfirmModal` を表示 |
| `activeBranchDisplay` | `input` / `change` | 枝番表示の直接編集。変更時に `doApplyActiveBranchDisplayValue()` を実行 |
| `btnPrevBranch` | `click` | `slidePrevBranch()` で前の枝番へ |
| `btnNextBranch` | `click` | `slideNextBranch()` で次の枝番へ |

### 3.5 枝番追加・削除

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `btnAddBranch` | `click` | 未保存変更があれば確認。なければ `branchAddModal` を開く |
| `branchAddModalOk` | `click` | 枝番入力バリデーション。既存枝番なら `branchExistsModal`。新規なら `renderBranchTabs()` でタブを追加 |
| `btnDeleteBranch` | `click` | `branchDeleteModal` を開く |
| `branchDeleteModalOk` | `click` | 現在の枝番を削除し、次の枝番へ切り替え |

### 3.6 枝番切り替え確認モーダル

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `branchSwitchConfirmModalRegister` | `click` | 登録確認後、`branchRegisterResultModal` を表示。登録完了後、枝番切り替えを実行 |
| `branchSwitchConfirmModalDiscard` | `click` | 変更を破棄して枝番切り替えを実行 |
| `branchSwitchConfirmModalCancel` / Backdrop | `click` | モーダルを閉じ、`pendingAction` をクリア |

### 3.7 地図・プレビュー関連

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `btnSvgOutput` | `click` | 地図出力（SVG）（実装は TODO） |
| `inputSvgLoad` | `change` | SVG ファイルを読み込み（FileReader） |
| `btnFullscreenEdit` | `click` | 全画面編集モード（`fullscreenEditOverlay`）を表示 |
| `btnCloseFullscreenEdit` | `click` | 全画面編集モードを閉じる |
| `btnPreview` | `click` | 全体プレビュー（実装は TODO） |
| `btnOutput` | `click` | 出力（PDF/PNG/SVG 選択）。（実装は TODO） |

### 3.8 登録

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `btnRegister` | `click` | `openRegisterConfirmModal()` で登録確認モーダルを表示 |
| `registerConfirmModalOk` | `click` | 登録処理実行後、`registerResultModal` を表示（TODO: API 呼び出し） |

### 3.9 ナビゲーション

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `linkBack` | `click` | `checkActuallyDirty()` で未保存変更があれば `branchSwitchConfirmModal`。なければ `history.back()` |
| `linkHeaderMenu` | `click` | 未保存変更があれば確認モーダル表示後、`menu.html` へ遷移 |

### 3.10 状態管理

| 変数・関数 | 用途 |
|------------|------|
| `dirty` | フォームの未保存変更フラグ |
| `originalBranchValues` | 枝番・備考の元の値（変更検知用） |
| `checkActuallyDirty()` | 枝番・備考の変更を検知 |
| `setDirty(value)` | `dirty` の更新 |
| `applyOrderInfoFromParams()` | URL パラメータから注文情報を反映 |
| `applyBranchTabsForOrder(orderNo)` | 枝番タブを注文に応じて更新 |

## 4. 注文 / 看板情報一覧（order_list.html）

### 4.1 検索条件

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `searchConditionToggle` | `click` | `searchConditionBody` の表示/非表示を切り替え |
| `btnSearch` | `click` | `performSearch()` で検索実行。注文番号でフィルタし、`orderListSection` を表示 |

### 4.2 検索・ページング

| 関数 | 制御内容 |
|------|----------|
| `performSearch()` | `inputOrderNo` で `tr[data-order-no]` をフィルタ。`matchedRows` を更新し、`renderPage()` と `renderPagination()` を実行 |
| `renderPage()` | 現在ページの行のみ表示（`display` で制御） |
| `renderPagination()` | 前後ボタン・ページ番号ボタンを生成。`currentPage` に応じて有効/無効を切り替え |

### 4.3 ページング

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `orderListPaginationPrev` | `click` | `currentPage--` し、`renderPage()` と `renderPagination('prev')` を実行 |
| `orderListPaginationNext` | `click` | `currentPage++` し、`renderPage()` と `renderPagination('next')` を実行 |
| ページ番号ボタン | `click` | `currentPage` を更新し、`renderPage()` と `renderPagination()` を実行 |

### 4.4 リンク生成（AppConfig 連携）

| 処理 | 制御内容 |
|------|----------|
| `td.col-action a[href*="order_detail"]` | `AppConfig.buildOrderParams()` でパラメータを構築。枝番（itemCode）を含む URL を生成 |
| `.order-main-link` | `AppConfig.buildOrderParams()` でパラメータを構築。`order_main.html` へのリンクを生成 |
| `attachListLinkClickHandler()` | クリック時に `AppConfig.replaceListUrlWithSearchState()` で URL に検索条件を反映 |

### 4.5 日付ピッカー

| 要素 | 制御内容 |
|------|----------|
| `inputUpdateDateFrom` / `inputUpdateDateTo` | flatpickr で日付選択（日本語ロケール、`Y/m/d` 形式） |

### 4.6 戻る

| 要素 | イベント | 制御内容 |
|------|----------|----------|
| `linkBack` | `click` | `history.back()` |

### 4.7 初期化・URL

| 処理 | 制御内容 |
|------|----------|
| 初期化 | URL の `orderNo` / `searched` があれば `performSearch()` を実行 |
| `pageshow` イベント | bfcache 復元時に `restoreFromUrlAndSearch()` を実行 |

## 5. 共通設定（js/app-config.js）

### 5.1 公開 API

| 関数 | 用途 |
|------|------|
| `buildOrderParams(source, extra, includeDetailKeys)` | `tr` 要素またはオブジェクトから `URLSearchParams` を構築。`ORDER_PARAM_KEYS` と `ORDER_DETAIL_EXTRA_KEYS` を使用 |
| `buildMainUrlFromCurrent(keys)` | 現在の URL から指定キーを抽出し、メイン URL を構築 |
| `onPageshowReplaceWithMainUrl(paramKeys)` | bfcache 復元時に URL を置換するイベントを登録 |
| `replaceListUrlWithSearchState(pathname, orderNo)` | 一覧画面の検索条件を URL に反映 |

### 5.2 パラメータキー

| 定数 | 内容 |
|------|------|
| `ORDER_PARAM_KEYS` | mode, orderNo, orderName, address, companyName, manager, template, designType, updateDate, updater |
| `ORDER_DETAIL_EXTRA_KEYS` | itemCode（枝番） |
| `LIST_PARAM_KEYS` | orderNo, searched |

## 6. 画面遷移フロー

```
menu.html
  ├─ order_main.html（注文情報入力 / 変更）
  │   ├─ order_detail.html（看板情報入力 / 変更）
  │   └─ order_list.html（注文 / 看板情報一覧）
  ├─ order_detail.html（看板情報入力 / 変更）
  └─ order_list.html（注文 / 看板情報一覧）
  │   ├─ order_main.html（注文番号クリック）
  │   └─ order_detail.html（枝番クリック）
```

## 7. モーダル一覧

| モーダル ID | 画面 | 用途 |
|-------------|------|------|
| `orderNoSelectModal` | order_main, order_detail | 注文番号選択 |
| `companySelectModal` | order_main | 会社選択 |
| `changeNoticeModal` | order_main | 新規→変更で入力変更がある場合の確認 |
| `unsavedChangesConfirmModal` | order_main | 未保存変更時の破棄確認 |
| `requiredValidationModal` | order_main | 必須項目未入力 |
| `registerConfirmModal` | order_main, order_detail | 登録/更新の確認 |
| `registerResultModal` | order_main, order_detail | 登録/更新の結果 |
| `orderSearchValidationModal` | order_detail | 検索時注文番号未入力 |
| `branchAddModal` | order_detail | 枝番追加 |
| `branchSwitchConfirmModal` | order_detail | 枝番切り替え時の未保存変更確認 |
| `branchRegisterResultModal` | order_detail | 枝番登録の結果 |
| `branchExistsModal` | order_detail | 既存枝番の重複 |
| `branchDeleteModal` | order_detail | 枝番削除確認 |
| `fullscreenEditOverlay` | order_detail | 全画面編集モード |
