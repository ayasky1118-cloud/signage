# 案2: フロントエンド構成整理 手順書

本ドキュメントは、案2「フロントエンドの整理」を実施するための手順である。
実施前に本ドキュメントを一通り確認し、作業の流れを把握してから進めること。

---

## 実施内容の概要

| 手順 | 内容 |
|------|------|
| 1 | ルートの package.json / node_modules を削除 |
| 2 | 不要ファイルの削除（router/index copy.ts、HelloWorld.vue） |

---

## 手順1: ルートの package.json / node_modules を削除

ルート直下の `package.json` と `node_modules` は、frontend 側に同様の依存関係があるため不要。
削除して依存関係を frontend に集約する。

### 1-1. 削除対象の確認

```bash
# map_pj のルートで実行
ls -la package.json node_modules 2>/dev/null || echo "ファイルが存在しません"
```

### 1-2. 削除実行

```bash
# map_pj のルートで実行
rm -rf package.json package-lock.json node_modules
```

### 1-3. 確認

- ルート直下に `package.json`、`package-lock.json`、`node_modules` が存在しないこと

---

## 手順2: 不要ファイルの削除

### 2-1. router/index copy.ts の削除

コピーファイルが残っているため削除する。

```bash
# map_pj のルートで実行
rm frontend/src/router/index\ copy.ts
```

### 2-2. HelloWorld.vue の削除

Vite のテンプレート由来のコンポーネント。App.vue やルーターで参照されていないため削除する。

```bash
# map_pj のルートで実行
rm frontend/src/components/HelloWorld.vue
```

### 2-3. 確認

- `frontend/src/router/` に `index copy.ts` が存在しないこと
- `frontend/src/components/HelloWorld.vue` が存在しないこと

---

## 手順3: 新規フォルダの作成

将来の拡張を見据えて、以下のフォルダを作成する。
現時点では空フォルダで問題ない（必要に応じて後からファイルを追加する）。

### 3-1. layouts フォルダ

共通レイアウト（ヘッダー・フッター・サイドバーなど）を配置するフォルダ。

```bash
mkdir -p frontend/src/layouts
```

**用途の例：**
- `DefaultLayout.vue` … 共通ヘッダー・フッター付きレイアウト
- `AuthLayout.vue` … ログイン画面用レイアウト

### 3-2. composables フォルダ

再利用可能なロジック（Composition API）を配置するフォルダ。

```bash
mkdir -p frontend/src/composables
```

**用途の例：**
- `useAuth.ts` … 認証状態の管理
- `useApi.ts` … API 呼び出しの共通処理

### 3-3. types フォルダ

TypeScript の型定義を配置するフォルダ。

```bash
mkdir -p frontend/src/types
```

**用途の例：**
- `api.ts` … API レスポンスの型定義
- `models.ts` … ドメインモデルの型定義

### 3-4. assets/images フォルダ

画像ファイルを配置するフォルダ（必要に応じて使用）。

```bash
mkdir -p frontend/src/assets/images
```

### 3-5. フォルダ構造の確認

```bash
# 作成後の frontend/src の構造を確認
find frontend/src -type d | sort
```

**期待される出力例：**

```
frontend/src
frontend/src/assets
frontend/src/assets/images
frontend/src/assets/styles
frontend/src/components
frontend/src/composables
frontend/src/layouts
frontend/src/pages
frontend/src/router
frontend/src/types
```

## 手順4: 動作確認

### 4-1. 開発サーバー起動

```bash
cd frontend

```

- `/menu` でメニュー画面が表示されること
- 他ページ（/order/main、/order/list など）も正常に表示されること

```bash
cd frontend
npm run build
```

エラーなくビルドが完了すること。

## トラブルシューティング

### ビルドエラーが出る

- `frontend/node_modules` が正しく存在するか確認
- `cd frontend && npm install` を実行して依存関係を再インストール

### ルートで npm コマンドを使っていた場合

- 今後は必ず `cd frontend` してから `npm run dev` 等を実行する
- ドキュメント（0〜6）の手順も、frontend ディレクトリを基準としている
