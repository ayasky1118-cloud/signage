# Amplify デプロイ手順

フロントエンドを Amplify にデプロイする手順。

---

## 前提

- Node.js / npm がインストール済み
- AWS CLI が設定済み
- API Gateway の URL が確定している
- **CGS 環境用**: Cognito User Pool を新規作成済み（[06b.Cognito作成手順.md](06b.Cognito作成手順.md) 参照）
- **手動デプロイ時**: `frontend/.env` および `.env.staging` に環境変数を設定済み
- **GitLab CI デプロイ時**: GitLab CI/CD 変数に上記を設定済み。Runner（`signage_dev_runner` タグ）が稼働していること

---

## 1. スクリプトでデプロイ（推奨）

```bash
cd /Users/yoshimi/dev/signage
./scripts/deploy-amplify.sh
```

### 1.1 スクリプトの動作

| 項目 | 値 |
|------|-----|
| ビルドコマンド | `npm run build:staging`（DEPLOY_ENV 未指定時） |
| Amplify アプリ ID | d27nzen1vhtcee |
| ブランチ名 | main |
| リージョン | ap-northeast-1（東京） |

### 1.2 環境別デプロイ

```bash
# 検証環境（デフォルト）
./scripts/deploy-amplify.sh

# 本番環境
DEPLOY_ENV=production ./scripts/deploy-amplify.sh
```

※ 本番時は `frontend/.env.production` に `VITE_API_BASE` 等を設定すること。デフォルトは `.env.staging` を使用。

## 2. 手動デプロイ

```bash
cd frontend
npm ci
npm run build:staging

cd dist
zip -r ../build.zip .
cd ..

aws amplify create-deployment \
  --app-id d27nzen1vhtcee \
  --branch-name main \
  --region ap-northeast-1 \
  --output json > deploy_info.json

UPLOAD_URL=$(python3 -c "import json; print(json.load(open('deploy_info.json'))['zipUploadUrl'])")
JOB_ID=$(python3 -c "import json; print(json.load(open('deploy_info.json'))['jobId'])")

curl -X PUT --upload-file build.zip "$UPLOAD_URL"

aws amplify start-deployment \
  --app-id d27nzen1vhtcee \
  --branch-name main \
  --job-id "$JOB_ID" \
  --region ap-northeast-1
```

## 3. 環境変数

### 3.1 ファイル一覧

| ファイル | 用途 |
|----------|------|
| .env | 全環境共通（MapTiler, Cognito） |
| .env.staging | 検証環境（VITE_API_BASE=API Gateway URL） |
| .env.production | 本番環境 |

※ いずれも `frontend/` 直下に配置。`.gitignore` によりリポジトリに含めない。

### 3.2 手動デプロイ時の設定例

**frontend/.env**（全環境共通）:

```
# MapTiler API キー（地図表示用）
VITE_MAPTILER_API_KEY=oB8ILFlAMMLB99O7Iqaa

# AWS Cognito（CGS 環境用）
VITE_COGNITO_USER_POOL_ID=ap-northeast-1_1jxjm5adQ
VITE_COGNITO_USER_POOL_CLIENT_ID=4uh1iskbs8plllsvvhigf8slce
```

**frontend/.env.staging**（検証環境）:

```
# API Gateway エンドポイント
VITE_API_BASE=https://tpdeco04lg.execute-api.ap-northeast-1.amazonaws.com/prod
```

※ `npm run build:staging` 実行時、`.env` と `.env.staging` がマージされて読み込まれる。

## 5. GitLab CI 変数

main への push で GitLab CI がビルド・デプロイする場合、`.env.staging` はリポジトリに含まれないため、**GitLab CI/CD 変数**で以下を設定する必要がある。

**設定場所**: GitLab プロジェクト → **設定** → **CI/CD** → **変数** → **変数を追加**

### 5.1 変数一覧

| キー | 値 | マスク |
|------|-----|--------|
| `VITE_API_BASE` | `https://tpdeco04lg.execute-api.ap-northeast-1.amazonaws.com/prod` | いいえ |
| `VITE_COGNITO_USER_POOL_ID` | `ap-northeast-1_1jxjm5adQ` | いいえ |
| `VITE_COGNITO_USER_POOL_CLIENT_ID` | `4uh1iskbs8plllsvvhigf8slce` | いいえ |
| `VITE_MAPTILER_API_KEY` | MapTiler で発行した API キー（例: `oB8ILFlAMMLB99O7Iqaa`） | はい（推奨） |

※ CI は `npm run build:staging` を実行し、上記変数をビルド時に読み込む。

### 5.2 各変数のフォーム設定値

「変数を追加」画面で、各項目を次のように設定する。

| 項目 | 設定値 |
|------|--------|
| **タイプ** | 変数（デフォルト） |
| **環境** | すべて（デフォルト） |
| **可視化** | 表示（VITE_MAPTILER_API_KEY のみ「マスクする」を推奨） |
| **フラグ** | 変数の保護: オフ、変数参照を展開: オン |
| **説明** | （空欄で可） |

#### VITE_API_BASE

| 項目 | 値 |
|------|-----|
| キー | `VITE_API_BASE` |
| 値 | `https://tpdeco04lg.execute-api.ap-northeast-1.amazonaws.com/prod` |

#### VITE_COGNITO_USER_POOL_ID

| 項目 | 値 |
|------|-----|
| キー | `VITE_COGNITO_USER_POOL_ID` |
| 値 | `ap-northeast-1_1jxjm5adQ` |

#### VITE_COGNITO_USER_POOL_CLIENT_ID

| 項目 | 値 |
|------|-----|
| キー | `VITE_COGNITO_USER_POOL_CLIENT_ID` |
| 値 | `4uh1iskbs8plllsvvhigf8slce` |

#### VITE_MAPTILER_API_KEY

| 項目 | 値 |
|------|-----|
| キー | `VITE_MAPTILER_API_KEY` |
| 値 | MapTiler で発行した API キー |
| 可視化 | **マスクする**（API キー保護のため推奨） |

### 5.3 CI の設定（.gitlab-ci.yml）

`build_site` ジョブで `npm run build:staging` を実行し、上記変数をビルド時に読み込む。Runner タグは `signage_dev_runner`。
