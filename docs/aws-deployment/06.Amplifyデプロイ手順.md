# Amplify デプロイ手順

フロントエンドを Amplify にデプロイする手順。

---

## 前提

- Node.js / npm がインストール済み
- AWS CLI が設定済み
- API Gateway の URL が確定している
- **CGS 環境用**: Cognito User Pool を新規作成済み（[06b.Cognito作成手順.md](06b.Cognito作成手順.md) 参照）。`frontend/.env` または `.env.staging` に `VITE_COGNITO_USER_POOL_ID` と `VITE_COGNITO_USER_POOL_CLIENT_ID` を設定すること

---

## 1. スクリプトでデプロイ（推奨）

```bash
cd /Users/yoshimi/dev/signage
./scripts/deploy-amplify.sh
```

デフォルトで `npm run build:staging` を実行し、`.env.staging` の `VITE_API_BASE` を使用する。

---

## 2. 手動デプロイ

```bash
cd frontend
npm ci
npm run build:staging

cd dist
zip -r ../build.zip .
cd ..

aws amplify create-deployment \
  --app-id d27nzen1vhtcee \
  --branch-name main \
  --region ap-northeast-1 \
  --output json > deploy_info.json

UPLOAD_URL=$(python3 -c "import json; print(json.load(open('deploy_info.json'))['zipUploadUrl'])")
JOB_ID=$(python3 -c "import json; print(json.load(open('deploy_info.json'))['jobId'])")

curl -X PUT --upload-file build.zip "$UPLOAD_URL"

aws amplify start-deployment \
  --app-id d27nzen1vhtcee \
  --branch-name main \
  --job-id "$JOB_ID" \
  --region ap-northeast-1
```

---

## 3. 環境変数

| ファイル | 用途 |
|----------|------|
| .env | 全環境共通（MapTiler, Cognito） |
| .env.staging | 検証環境（VITE_API_BASE=API Gateway URL） |
| .env.production | 本番環境 |

---

## 4. デプロイ先

| 環境 | コマンド | URL |
|------|----------|-----|
| 検証 | `./scripts/deploy-amplify.sh` | https://main.d27nzen1vhtcee.amplifyapp.com |
| 本番 | `DEPLOY_ENV=production ./scripts/deploy-amplify.sh` | 本番 Amplify URL |
