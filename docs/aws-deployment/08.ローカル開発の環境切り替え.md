# ローカル開発の環境切り替え

ローカル・検証・本番の 3 環境で、設定を切り替える方法。

---

## 環境一覧

| 環境 | コマンド | API 接続先 |
|------|----------|------------|
| **ローカル** | `npm run dev` | Vite プロキシ → localhost:8000 |
| **検証** | `npm run build:staging` | 検証用 API Gateway |
| **本番** | `npm run build` / `npm run build:production` | 本番 API Gateway |

---

## フロントエンド

### ファイル構成

```
frontend/
├── .env                    # 全環境共通（MapTiler, Cognito）
├── .env.development        # ローカル用（VITE_API_BASE なし）
├── .env.staging            # 検証用（VITE_API_BASE=検証URL）
└── .env.production         # 本番用（VITE_API_BASE=本番URL）
```

### ローカル開発時

`npm run dev` 実行時は `.env.development` が自動読み込み。`VITE_API_BASE` は設定しないため、Vite プロキシ `/api` 経由で `localhost:8000` に転送される。

---

## バックエンド

### ローカル開発時

```bash
# backend/.env
DATABASE_URL=mysql+pymysql://root:password@db:3306/signage_dev?charset=utf8mb4
```

`docker compose up -d backend` の場合は docker-compose の environment を使用。

### Lambda

Lambda の環境変数で `DATABASE_URL` に RDS Proxy の URL を設定。

---

## 起動手順（ローカル開発）

```bash
cd /Users/yoshimi/dev/signage

# 1. DB 起動
docker compose up -d db

# 2. 初回のみ: DB スキーマ投入
./db/apply_schema.sh

# 3. バックエンド起動
docker compose up -d backend

# 4. フロントエンド起動
cd frontend && npm run dev
```
