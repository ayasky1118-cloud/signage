# 01. Dockerfile 作成手順

フロントエンド（Vue）とバックエンド（FastAPI）の Dockerfile を作成し、ローカルでの動作確認に利用する。

---

## 用途

| デプロイ先 | フロントエンド | バックエンド |
|------------|----------------|--------------|
| **Amplify** | `npm run build` でビルドし直接デプロイ。Dockerfile は使わない | - |
| **Lambda** | - | ZIP（handler.py + 依存関係）でデプロイ。Dockerfile は使わない |
| **ローカル動作確認** | Dockerfile でビルド・起動可能 | docker-compose で backend を起動（Dockerfile 使用） |

---

## 前提

- Docker がインストール済みであること
- プロジェクトルート: `signage/`

---

## 作成するファイル

| ファイル | 説明 |
|----------|------|
| `frontend/Dockerfile` | フロントエンド用（ビルド → nginx で静的配信）。ローカル確認用 |
| `backend/Dockerfile` | バックエンド用（FastAPI + uvicorn）。docker-compose 用 |

---

## 1. フロントエンド Dockerfile

### 配置場所

```
signage/frontend/Dockerfile
```

### 構成

- **マルチステージビルド**
  - Stage 1: Node.js で `npm run build` を実行
  - Stage 2: nginx で `dist/` を静的配信

### ビルド時の環境変数

フロントエンドはビルド時に `VITE_*` 変数を埋め込む。必要に応じて `docker build --build-arg` で渡す。

| 変数 | 説明 |
|------|------|
| `VITE_API_BASE` | バックエンド API のベース URL（Amplify の場合は API Gateway の URL） |
| `VITE_MAPTILER_API_KEY` | MapTiler 地図表示用 |
| `VITE_COGNITO_USER_POOL_ID` | Cognito User Pool ID |
| `VITE_COGNITO_USER_POOL_CLIENT_ID` | Cognito App Client ID |

### 動作確認

```bash
cd signage/frontend
docker build -t signage-frontend .
docker run -p 8080:80 signage-frontend
# ブラウザで http://localhost:8080 を開く
```

---

## 2. バックエンド Dockerfile

### 配置場所

```
signage/backend/Dockerfile
```

### 構成

- Python 3.12 ベース
- `requirements.txt` で依存関係をインストール
- uvicorn で FastAPI を起動（ポート 8000）

### 実行時の環境変数

本番ではコンテナ起動時に環境変数で渡す。

| 変数 | 説明 |
|------|------|
| `DATABASE_URL` | MySQL 接続 URL（RDS 等） |
| `MAPTILER_API_KEY` | 住所検証・ジオコーディング用（任意） |

### 動作確認（推奨: docker compose）

`docker-compose.yml` に backend サービスを追加済み。db と同一ネットワークで `db:3306` に接続する。

```bash
cd signage
docker compose up -d db    # MySQL が未起動の場合
docker compose up -d backend
# http://localhost:8000/health で確認
```

### 動作確認（docker run で単体実行）

```bash
cd signage/backend
docker build -t signage-backend .
docker run -p 8000:8000 \
  -e DATABASE_URL="mysql+pymysql://root:password@host.docker.internal:3307/signage_dev?charset=utf8mb4" \
  signage-backend
# http://localhost:8000/health で確認
```

**注意（docker run 時）**:
- ホストの MySQL に接続するには `host.docker.internal` を使用（Mac/Windows Docker Desktop）
- Linux の場合は `--add-host=host.docker.internal:host-gateway` を追加
- docker-compose の MySQL はポート 3307 で公開されているため、`:3307` を指定

---

## 3. トラブルシューティング（バックエンド）

| 現象 | 原因 | 対処 |
|------|------|------|
| コンテナ起動後すぐ終了 | `DATABASE_URL` 未設定、または接続先がコンテナから到達不可 | `docker compose up` を使う（db と同一ネットワーク）か、`host.docker.internal` を指定 |
| `Connection refused` | MySQL が未起動、またはホスト名・ポートが誤り | `docker compose up -d db` で MySQL を起動。docker run の場合は `host.docker.internal:3307` |
| Linux で `host.docker.internal` が解決できない | Linux ではデフォルトで未定義 | `docker run` に `--add-host=host.docker.internal:host-gateway` を追加 |
| ポート 8000 が使用中 | 既存のプロセスが占有 | `docker compose down` または該当プロセスを停止 |

---

## 4. 依存関係の確認

バックエンドの `config.py` は `python-dotenv` と `pydantic-settings` を使用している。  
`requirements.txt` に含まれていない場合は追加する。

```txt
python-dotenv
pydantic-settings
```
