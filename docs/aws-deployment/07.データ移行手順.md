# データ移行手順

ローカル Docker の MySQL と AWS RDS の間でデータを移行・復元する手順。

---

## 前提

- ローカル: Docker で MySQL が起動している
- ローカル: `mysql` および `mysqldump` クライアントがインストール済み
- RDS: パブリックアクセス「あり」
- RDS: セキュリティグループでマイ IP から 3306 を許可済み

---

## エンドポイントの使い分け

| エンドポイント | 用途 | ローカルから接続 |
|----------------|------|------------------|
| signage-dev-db.xxx.rds.amazonaws.com | スキーマ投入・データ移行・復元 | ◎ 可能 |
| signage-dev-proxy.proxy-xxx.rds.amazonaws.com | Lambda からの接続 | × 不可（VPC 内のみ） |

**データ移行・復元では RDS インスタンスのエンドポイントを使用する。**

---

## 1. ローカル → RDS（データ移行）

### 1.1 初回: スキーマ投入

RDS が空の場合は、スキーマを投入する:

```bash
cd /Users/yoshimi/dev/signage
export RDS_HOST="signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com"
export RDS_USER="admin" RDS_PASSWORD="<パスワード>"
for f in db/00_schema/*.sql; do
  mysql -h"$RDS_HOST" -u"$RDS_USER" -p"$RDS_PASSWORD" --ssl-mode=REQUIRED signage_dev < "$f"
done
```

### 1.2 移行実行

```bash
RDS_HOST="signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com" \
RDS_USER="admin" RDS_PASSWORD="<パスワード>" \
./scripts/migrate-to-rds.sh signage_dev
```

※ RDS 側の既存テーブルは**上書き**される。事前にバックアップすること。

---

## 2. RDS → ローカル（復元）

```bash
RDS_HOST="signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com" \
RDS_USER="admin" RDS_PASSWORD="<パスワード>" \
./scripts/migrate-from-rds.sh signage_dev
```

※ ローカルの既存データは**上書き**される。確認プロンプトが表示される。

---

## 3. 接続エラー時

- **タイムアウト**: セキュリティグループでマイ IP を許可しているか確認。`./scripts/update-db-sg-my-ip.sh` を実行
- **ルートテーブル**: RDS のサブネットが IGW 経由であること（[02.VPC・ネットワーク構築手順.md](02.VPC・ネットワーク構築手順.md) 参照）
