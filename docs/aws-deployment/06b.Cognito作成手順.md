# Cognito 作成手順（CGS 環境用）

CGS 環境で認証に使用する Cognito User Pool の作成手順。

※ 従来の sinage_old テスト環境の User Pool（ap-northeast-1_YCSDgFpIR）とは別に、CGS 環境用を新規作成する必要がある。

---

## 0. スクリプトで作成（推奨）

```bash
cd /Users/yoshimi/dev/signage
./scripts/create-cognito-user-pool.sh
```

作成後、出力された ID を `frontend/.env` または `frontend/.env.staging` に設定する。

---

## 1. 手動でユーザープールの作成（コンソール）

1. **AWS コンソール** → **Cognito** → **ユーザープール** → **ユーザープールの作成**
2. **認証プロバイダー**: Cognito ユーザープール
3. **Cognito ユーザープールのサインインオプション**: メールアドレス（またはユーザー名）
4. **パスワードポリシー**: 任意（推奨: 8 文字以上）
5. **ユーザープール名**: `signage-cgs-pool`（任意）
6. **アプリクライアント**: 作成する
   - アプリクライアント名: `signage-web`
   - クライアントシークレット: 生成しない（パブリッククライアント）
   - 認証フロー: ALLOW_USER_PASSWORD_AUTH, ALLOW_REFRESH_TOKEN_AUTH
7. **作成**

---

## 2. ID の確認

作成後、以下を控える:

| 項目 | 取得場所 |
|------|----------|
| ユーザープール ID | ユーザープール詳細 → ユーザープール ID（例: ap-northeast-1_xxxxxxxxx） |
| アプリクライアント ID | アプリの統合 → アプリクライアント → クライアント ID |

---

## 3. フロントエンドの設定

`frontend/.env` または `frontend/.env.staging` に以下を設定:

```
VITE_COGNITO_USER_POOL_ID=ap-northeast-1_xxxxxxxxx
VITE_COGNITO_USER_POOL_CLIENT_ID=xxxxxxxxxxxxxxxxxxxxxxxxxx
```

※ `main.ts` は環境変数を読み込む。未設定時は従来のテスト環境の User Pool にフォールバックするため、**CGS デプロイ時は必ず設定すること**。

---

## 4. 現行構成リファレンスの更新

作成後、[01.現行構成リファレンス.md](01.現行構成リファレンス.md) の Cognito セクションに User Pool ID とアプリクライアント ID を記載する。

---

## 5. auth_uid の設定

ログイン可能にするには、`user` テーブルの `auth_uid` を Cognito の **sub** と一致させる必要がある。

1. 一度ログインを試す → 「ログインできません」画面に表示される Cognito sub をコピー
2. RDS の `user` テーブルを更新:
   ```sql
   UPDATE user SET auth_uid = 'Cognitoのsubをここに貼り付け' WHERE user_id = 1;
   ```

詳細は [07.データ移行手順.md](07.データ移行手順.md) の auth_uid 設定を参照。
