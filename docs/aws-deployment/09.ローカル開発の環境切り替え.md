# ローカル開発の環境切り替え

ローカル・検証・本番の 3 環境で、設定を切り替える方法。

---

## 前提（初回セットアップ）

09 を実行する前に、以下を済ませておく。

| 項目 | 手順 | 参照 |
|------|------|------|
| Docker のインストール | [00.環境準備手順.md](00.環境準備手順.md) | 別 PC 用の環境構築 |
| Node.js のインストール | [101.環境構築ステップ.md](../101.環境構築ステップ.md) | 必須ツール |
| frontend/.env の作成 | `.env.example` をコピーし、`VITE_MAPTILER_API_KEY` を設定（Cognito 利用時は `VITE_COGNITO_*` も） | [203.本番環境の環境変数.md](../203.本番環境の環境変数.md) |
| frontend の依存関係 | `cd frontend && npm install` | - |
| DB スキーマの投入（初回のみ） | `docker compose up -d db` の後、`./db/apply_schema.sh` を実行 | - |

※ `backend/.env` は `docker compose up -d backend` の場合は不要（docker-compose の environment を使用）。`uvicorn` を直接起動する場合のみ必要。

---

## 環境一覧

| 環境 | コマンド | 読み込むファイル | API 接続先 |
|------|----------|------------------|------------|
| **ローカル** | `npm run dev` | `.env.development` | Vite プロキシ → localhost:8000 |
| **検証** | `npm run build:staging` | `.env.staging` | 検証用 API Gateway |
| **本番** | `npm run build` / `npm run build:production` | `.env.production` | 本番 API Gateway |

**手動での切り替えは不要**。コマンド実行時に自動で適切な設定が読み込まれる。

---

## 切り替えが必要な項目

| 項目 | ローカル | 検証 | 本番 |
|------|----------|------|------|
| **API の接続先** | ローカルバックエンド（Vite プロキシ経由） | 検証用 API Gateway | 本番 API Gateway |
| **DB の接続先** | ローカル Docker MySQL | RDS / RDS Proxy（検証用） | RDS / RDS Proxy（本番） |

## フロントエンド（frontend/）

### ファイル構成

```
frontend/
├── .env                    # 全環境共通（MapTiler, Cognito など）
├── .env.development        # ローカル用（VITE_API_BASE なし）
├── .env.staging            # 検証用（VITE_API_BASE=検証URL）
└── .env.production         # 本番用（VITE_API_BASE=本番URL）
```

### ローカル開発時

`npm run dev` 実行時は `.env.development` が自動読み込みされる。`VITE_API_BASE` は設定しないため、Vite プロキシ `/api` 経由で `localhost:8000` に転送される。

### 検証・本番ビルド時

- 検証用: `npm run build:staging` → `.env.staging` を読み込み
- 本番用: `npm run build` または `npm run build:production` → `.env.production` を読み込み

### 検証環境の URL

現在構築済みの AWS は検証環境。`.env.staging` に API Gateway の URL を設定済み。

## デプロイスクリプト（scripts/deploy-amplify.sh）

現在構築済みの AWS（Amplify + Lambda + RDS）は**検証環境**。環境変数 `DEPLOY_ENV` でビルドモードを切り替え:

```bash
# 検証環境デプロイ（デフォルト）
./scripts/deploy-amplify.sh

# 本番デプロイ（本番 AWS 構築後）
DEPLOY_ENV=production ./scripts/deploy-amplify.sh
```

## バックエンド（backend/.env）

### ローカル開発時

```bash
# backend/.env（ローカル開発用）
# docker compose で backend を動かす場合
DATABASE_URL=mysql+pymysql://root:password@db:3306/signage_dev?charset=utf8mb4

# uvicorn を直接起動する場合（ホストから Docker の MySQL に接続）
DATABASE_URL=mysql+pymysql://root:password@127.0.0.1:3307/signage_dev?charset=utf8mb4
```

※ ポート 3307 は docker-compose で `3307:3306` とマッピングしているため。

### 本番（Lambda）

Lambda の環境変数で `DATABASE_URL` に RDS Proxy の URL を設定。`backend/.env` はデプロイに含まれない。

## 起動手順（ローカル開発）

```bash
cd /Users/yoshimi/dev/signage

# 1. DB 起動
docker compose up -d db

# 2. 初回のみ: DB スキーマ投入（テーブル作成・シード投入。2回目以降はスキップ）
./db/apply_schema.sh

# 3. バックエンド起動（どちらか）
docker compose up -d backend
# または
cd backend && source .venv/bin/activate && uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 4. フロントエンド起動
cd frontend && npm run dev
```

## チェックリスト（ローカル開発時）

| 確認項目 | ローカル開発時 |
|----------|----------------|
| `frontend/.env.development` に `VITE_API_BASE` が**ない**（または空） | ☐ |
| `backend/.env` の `DATABASE_URL` がローカル DB を指している | ☐ |
| Docker MySQL が起動している（`docker compose up -d db`） | ☐ |
| バックエンドが `localhost:8000` で起動している | ☐ |
