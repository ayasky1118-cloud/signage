# VPC・ネットワーク構築手順

現行構成に基づく、VPC・サブネット・ルートテーブル・NAT ゲートウェイの構築手順。

---

## 前提

- デフォルト VPC を使用する場合は、本手順の多くはスキップ可能
- カスタム VPC を新規作成する場合は、本手順に従って構築する

---

## アーキテクチャ概要

| 用途 | サブネット | ルート | 備考 |
|------|------------|--------|------|
| RDS（DBeaver 接続） | IGW 経由のサブネット | 0.0.0.0/0 → IGW | インターネットからの接続を許可 |
| Lambda（MapTiler API） | NAT 経由のサブネット | 0.0.0.0/0 → NAT | 住所検証・ジオコーディング用 |
| NAT ゲートウェイ | パブリックサブネット | 0.0.0.0/0 → IGW | NAT 本体の配置先 |

---

## 1. デフォルト VPC を使用する場合

### 1.1 VPC の確認

```bash
aws ec2 describe-vpcs --filters "Name=is-default,Values=true" --region ap-northeast-1
```

### 1.2 サブネットの確認

```bash
aws ec2 describe-subnets --filters "Name=vpc-id,Values=vpc-030cd3d1451ebff97" --region ap-northeast-1
```

---

## 2. NAT ゲートウェイの作成

Lambda が MapTiler API にアクセスするために必要。

### 2.1 パブリックサブネットの確認

NAT ゲートウェイは IGW 経由のサブネット（パブリックサブネット）に配置する。

現行: subnet-07b3bd645bcac0035（ap-northeast-1a）

### 2.2 NAT ゲートウェイの作成

1. **EC2** → **NAT ゲートウェイ** → **NAT ゲートウェイを作成**
2. **サブネット**: パブリックサブネット（例: subnet-07b3bd645bcac0035）を選択
3. **Elastic IP**: 新規割り当てを選択
4. **作成**

```bash
# Elastic IP を割り当て
ALLOC_ID=$(aws ec2 allocate-address --domain vpc --region ap-northeast-1 --query "AllocationId" --output text)

# NAT ゲートウェイ作成
aws ec2 create-nat-gateway \
  --subnet-id subnet-07b3bd645bcac0035 \
  --allocation-id $ALLOC_ID \
  --region ap-northeast-1
```

---

## 3. Lambda 用サブネットの作成（NAT 経由）

デフォルト VPC に Lambda 用のサブネットを追加する場合。

### 3.1 サブネット作成

```bash
aws ec2 create-subnet \
  --vpc-id vpc-030cd3d1451ebff97 \
  --cidr-block 172.31.48.0/20 \
  --availability-zone ap-northeast-1a \
  --region ap-northeast-1 \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=signage-lambda-nat}]'
```

### 3.2 NAT ルートテーブルの作成

```bash
# ルートテーブル作成
RT_ID=$(aws ec2 create-route-table \
  --vpc-id vpc-030cd3d1451ebff97 \
  --region ap-northeast-1 \
  --tag-specifications 'ResourceType=route-table,Tags=[{Key=Name,Value=signage-lambda-nat-rt}]' \
  --query "RouteTable.RouteTableId" --output text)

# NAT ルートを追加
aws ec2 create-route \
  --route-table-id $RT_ID \
  --destination-cidr-block 0.0.0.0/0 \
  --nat-gateway-id nat-0e90841e68eb923a6 \
  --region ap-northeast-1

# サブネットを関連付け
aws ec2 associate-route-table \
  --route-table-id $RT_ID \
  --subnet-id <新規サブネットID> \
  --region ap-northeast-1
```

## 4. RDS 用サブネットのルート（IGW）

RDS を DBeaver から接続するには、RDS が配置されるサブネットのルートが IGW である必要がある。

### 4.1 サブネットのルートテーブル確認

RDS が配置されるサブネット（例: subnet-08acbf1c74b943f8f）のルートテーブルで、`0.0.0.0/0` が **インターネットゲートウェイ** を指していることを確認。

### 4.2 メインルートテーブル（IGW）への関連付け

サブネットが NAT ルートテーブルに紐づいている場合、メインルートテーブル（IGW）に切り替える:

```bash
# 関連付け ID を取得
aws ec2 describe-route-tables --route-table-ids rtb-09783c9a883d7ea07 --region ap-northeast-1 \
  --query "RouteTables[0].Associations[?SubnetId=='subnet-08acbf1c74b943f8f'].RouteTableAssociationId" --output text

# メインルートテーブルに切り替え
aws ec2 replace-route-table-association \
  --association-id <上記で取得したID> \
  --route-table-id rtb-0833af695c2db7099 \
  --region ap-northeast-1
```

---

## 5. 構成確認

```bash
# ルートテーブル一覧
aws ec2 describe-route-tables --filters "Name=vpc-id,Values=vpc-030cd3d1451ebff97" --region ap-northeast-1

# NAT ゲートウェイ状態
aws ec2 describe-nat-gateways --filter "Name=vpc-id,Values=vpc-030cd3d1451ebff97" --region ap-northeast-1
```
