# RDS 作成手順

RDS for MySQL の作成手順。現行構成に基づく。

---

## 前提

- VPC・サブネットが準備済み
- RDS を DBeaver から接続する場合は、RDS が配置されるサブネットのルートが IGW であること（[02.VPC・ネットワーク構築手順.md](02.VPC・ネットワーク構築手順.md) 参照）

---

## 1. データベースの作成

### 1.1 コンソールでの作成

1. **RDS** → **データベース** → **データベースを作成**
2. **エンジン**: MySQL 8.0
3. **テンプレート**: 開発/テスト
4. **設定**:
   - DB 識別子: `signage-dev-db`
   - マスターユーザー: `admin`
   - パスワード: 任意（控える）
5. **インスタンス**: db.t3.micro
6. **ストレージ**: 20 GB（gp3）
7. **接続**:
   - VPC: デフォルト VPC
   - パブリックアクセス: **あり**（DBeaver・データ移行用）
   - セキュリティグループ: 新規作成 `signage-db-sg`
8. **作成**

### 1.2 作成後のエンドポイント

```
signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com:3306
```

---

## 2. セキュリティグループの設定

### 2.1 必須ルール

**signage-db-sg** のインバウンドに以下を追加:

| タイプ | ポート | ソース | 用途 |
|--------|--------|--------|------|
| MySQL/Aurora | 3306 | マイ IP | DBeaver・データ移行 |
| MySQL/Aurora | 3306 | sg-04ebd02d056405e61（自己参照） | Lambda 直接接続時 |
| MySQL/Aurora | 3306 | sg-09651f78bc62d327c（RDS Proxy） | RDS Proxy → RDS |

### 2.2 マイ IP の更新

IP が変わって接続できなくなった場合:

```bash
./scripts/update-db-sg-my-ip.sh
```

### 2.3 手動でルール追加

1. **EC2** → **セキュリティグループ** → `signage-db-sg`
2. **インバウンドルールを編集** → **ルールを追加**
3. タイプ: MySQL/Aurora、ポート: 3306、ソース: マイ IP または カスタム（SG ID）

---

## 3. 接続確認

```bash
# 接続テスト
nc -zv signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com 3306

# MySQL クライアント
mysql -h signage-dev-db.cly840668ik8.ap-northeast-1.rds.amazonaws.com -P 3306 -u admin -p --ssl-mode=REQUIRED signage_dev -e "SELECT 1"
```
