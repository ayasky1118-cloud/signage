## 初回構築の順序

1. **環境準備**（00）: AWS CLI、Docker
2. **VPC・ネットワーク**（02）: NAT ゲートウェイ、Lambda 用サブネット
3. **RDS**（03）: signage-dev-db、signage-db-sg、マイ IP で 3306 許可
4. **Secrets Manager**（04）: signage-dev/db-credentials
5. **RDS Proxy**（04）: signage-dev-proxy
6. **【必須】RDS の SG に RDS Proxy の SG を許可**（04）
7. **Lambda**（05）: VPC、NAT サブネット、環境変数
8. **API Gateway**（05）: HTTP API、CORS
9. **スキーマ投入**: RDS に db/00_schema/ を実行
10. **データ移行**（07）: migrate-to-rds.sh
11. **auth_uid 設定**: user テーブルの auth_uid を Cognito sub に合わせる
12. **GitLab CI 変数**（06）: main の push で CI デプロイする場合、GitLab → 設定 → CI/CD → 変数 に `VITE_API_BASE`, `VITE_COGNITO_USER_POOL_ID`, `VITE_COGNITO_USER_POOL_CLIENT_ID`, `VITE_MAPTILER_API_KEY` を追加

## Lambda デプロイ（コード変更時）

```bash
cd backend
rm -rf package && mkdir package
pip3 install -r requirements.txt -t package/ \
  --platform manylinux2014_x86_64 --python-version 3.12 --only-binary=:all:
cp -r app package/ && cp handler.py package/
cd package && zip -r ../lambda-deploy.zip . -x "*.pyc" -x "*__pycache__*" && cd ..
aws lambda update-function-code \
  --function-name signage-dev-api \
  --zip-file fileb://lambda-deploy.zip \
  --region ap-northeast-1 --no-cli-pager
```

## セキュリティグループの必須ルール

| 接続元 | 接続先 | ルール |
|--------|--------|--------|
| Lambda | RDS Proxy | RDS Proxy の SG で Lambda の SG からの 3306 を許可 |
| RDS Proxy | RDS | **RDS の SG で RDS Proxy の SG からの 3306 を許可**（必須） |
| ローカル | RDS | RDS の SG でマイ IP からの 3306 を許可 |

※ IP が変わった場合: `./scripts/update-db-sg-my-ip.sh`
