# Amplify 手動デプロイ手順

GitLab CI の Runner が利用できない場合、ローカルから Amplify に直接デプロイする手順。

---

## 前提

- Node.js / npm がインストール済み
- AWS CLI がインストール・設定済み（`aws configure` または環境変数で認証情報を設定）
- Amplify アプリへのデプロイ権限がある IAM ユーザー/ロール

---

## 手順

### 方法 1: スクリプトで一括実行（推奨）

```bash
cd /Users/yoshimi/dev/signage
chmod +x scripts/deploy-amplify.sh
./scripts/deploy-amplify.sh
```

### 方法 2: 手動で実行

```bash
cd /Users/yoshimi/dev/signage/frontend

# 1. ビルド
npm ci
npm run build

# 2. ZIP 作成
cd dist
zip -r ../build.zip .
cd ..

# 3. デプロイ（環境変数 VITE_API_BASE は GitLab CI 変数と同様に .env 等で設定）
aws amplify create-deployment \
  --app-id d27nzen1vhtcee \
  --branch-name main \
  --region ap-northeast-1 \
  --output json > deploy_info.json

UPLOAD_URL=$(python3 -c "import json; print(json.load(open('deploy_info.json'))['zipUploadUrl'])")
JOB_ID=$(python3 -c "import json; print(json.load(open('deploy_info.json'))['jobId'])")

curl -X PUT --upload-file build.zip "$UPLOAD_URL"

aws amplify start-deployment \
  --app-id d27nzen1vhtcee \
  --branch-name main \
  --job-id "$JOB_ID" \
  --region ap-northeast-1
```

---

## 環境変数について

スクリプト実行時、`VITE_API_BASE` はデフォルトで本番 API URL が設定されます。上書きする場合：

```bash
export VITE_API_BASE="https://tpdeco04lg.execute-api.ap-northeast-1.amazonaws.com/prod"
./scripts/deploy-amplify.sh
```

Cognito や MapTiler など、その他の環境変数は `frontend/.env.production` に設定してください（GitLab CI 変数と同様）。
