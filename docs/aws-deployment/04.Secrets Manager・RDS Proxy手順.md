# Secrets Manager・RDS Proxy 手順

DB 認証情報の管理と RDS Proxy の作成手順。

---

## 前提

- RDS が作成済み
- [03.RDS作成手順.md](03.RDS作成手順.md) 完了

---

## 1. Secrets Manager

### 1.1 シークレットの作成

1. **Secrets Manager** → **新しいシークレットの保存**
2. **シークレットのタイプ**: その他のシークレットのタイプ
3. **キー/値**:
   - `username`: RDS のマスターユーザー名（admin）
   - `password`: RDS のマスターパスワード
4. **シークレット名**: `signage-dev/db-credentials`
5. **作成**

### 1.2 現行のシークレット

| 項目 | 値 |
|------|-----|
| 名前 | signage-dev/db-credentials |
| ARN | arn:aws:secretsmanager:ap-northeast-1:805171368443:secret:signage-dev/db-credentials-* |

---

## 2. RDS Proxy

### 2.1 作成手順

1. **RDS** → **プロキシ** → **プロキシの作成**
2. **プロキシ識別子**: `signage-dev-proxy`
3. **エンジン**: MySQL
4. **認証**: Secrets Manager の `signage-dev/db-credentials` を選択
5. **サブネット**: RDS と同じ VPC のサブネット（複数 AZ を選択）
   - 現行: subnet-08acbf1c74b943f8f, subnet-07b3bd645bcac0035, subnet-07ab8df265549f7fe
6. **セキュリティグループ**: default（sg-09651f78bc62d327c）または新規
7. **ターゲットグループ**: RDS インスタンス `signage-dev-db` を選択
8. **作成**

### 2.2 エンドポイント

```
signage-dev-proxy.proxy-cly840668ik8.ap-northeast-1.rds.amazonaws.com:3306
```

### 2.3 【必須】RDS の SG に RDS Proxy を許可

RDS Proxy が RDS に接続するため、**RDS のセキュリティグループ**（signage-db-sg）のインバウンドに、**RDS Proxy の SG** からの 3306 を追加する。

```bash
aws ec2 authorize-security-group-ingress \
  --group-id sg-04ebd02d056405e61 \
  --protocol tcp --port 3306 \
  --source-group sg-09651f78bc62d327c \
  --region ap-northeast-1
```

※ 忘れると RDS Proxy ターゲットが UNAVAILABLE、Lambda が 500 エラーになる。

---

## 3. 接続の使い分け

| エンドポイント | 用途 | 接続元 |
|----------------|------|--------|
| signage-dev-db.xxx.rds.amazonaws.com | スキーマ投入・データ移行・DBeaver | ローカル |
| signage-dev-proxy.proxy-xxx.rds.amazonaws.com | Lambda からの接続 | Lambda（VPC 内のみ） |
