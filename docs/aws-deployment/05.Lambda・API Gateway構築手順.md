# Lambda・API Gateway 構築手順

バックエンド（FastAPI + Mangum）の Lambda と API Gateway の構築手順。

---

## 前提

- RDS Proxy が作成済み
- [04.Secrets Manager・RDS Proxy手順.md](04.Secrets%20Manager・RDS%20Proxy手順.md) 完了
- Lambda 用サブネット（NAT 経由）が準備済み

---

## 1. Lambda の作成

### 1.1 関数の作成

1. **Lambda** → **関数の作成**
2. **関数名**: `signage-dev-api`
3. **ランタイム**: Python 3.12
4. **アーキテクチャ**: x86_64
5. **作成**

### 1.2 VPC 設定

1. **設定** → **VPC** → **編集**
2. **VPC**: vpc-030cd3d1451ebff97（デフォルト VPC）
3. **サブネット**: **NAT 経由のサブネット**を選択
   - 現行: subnet-0ce8f2d34f7d880cd, subnet-07ab8df265549f7fe
4. **セキュリティグループ**: signage-db-sg（sg-04ebd02d056405e61）
5. **保存**

※ Lambda は MapTiler API にアクセスするため、**NAT 経由のサブネット**が必須。

### 1.3 環境変数

| キー | 値 |
|------|-----|
| API_PREFIX | /prod |
| DATABASE_URL | mysql+pymysql://admin:<パスワード>@signage-dev-proxy.proxy-cly840668ik8.ap-northeast-1.rds.amazonaws.com:3306/signage_dev?charset=utf8mb4 |
| MAPTILER_API_KEY | MapTiler で発行した API キー |
| CORS_ORIGINS | https://main.d27nzen1vhtcee.amplifyapp.com |

### 1.4 タイムアウト・メモリ

- タイムアウト: 30 秒
- メモリ: 512 MB

### 1.5 コードのデプロイ

```bash
cd backend
rm -rf package && mkdir package
pip3 install -r requirements.txt -t package/ \
  --platform manylinux2014_x86_64 --python-version 3.12 --only-binary=:all:
cp -r app package/ && cp handler.py package/
cd package && zip -r ../lambda-deploy.zip . -x "*.pyc" -x "*__pycache__*" && cd ..
aws lambda update-function-code \
  --function-name signage-dev-api \
  --zip-file fileb://lambda-deploy.zip \
  --region ap-northeast-1 --no-cli-pager
```

## 2. API Gateway

### 2.1 HTTP API の作成

1. **API Gateway** → **API の作成** → **HTTP API**
2. **統合**: Lambda
3. **Lambda 関数**: signage-dev-api
4. **API 名**: signage-dev-api
5. **作成**

### 2.2 ルートの設定

- `ANY /{proxy+}` → Lambda 統合
- `ANY /` → Lambda 統合

### 2.3 ステージ

- ステージ名: prod
- 自動デプロイ: 有効

### 2.4 エンドポイント

```
https://tpdeco04lg.execute-api.ap-northeast-1.amazonaws.com
```

### 2.5 CORS

- アクセス制御を許可する: 有効
- 許可するオリジン: Amplify の URL
