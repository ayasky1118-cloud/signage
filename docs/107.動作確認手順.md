# 動作確認手順（Vite + Vue3 + Vue Router）

本ドキュメントは、開発環境の動作確認を一括で行うための手順である。
各導入手順（101〜108）を実施した後に参照する。

---

## 0. 前提（API 利用時のみ）

API や DB 接続を確認する場合は、事前に以下を起動する。

### MySQL 起動

```bash
cd /path/to/signage
docker compose up -d
```

### バックエンド起動

```bash
cd /path/to/signage/backend
source .venv/bin/activate   # Windows: .venv\Scripts\activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

- API: `http://localhost:8000`
- ヘルスチェック: `GET /health`
- DB疎通確認: `GET /companies`

---

## 1. 開発サーバー起動

```bash
cd /path/to/frontend
npm run dev
```

---

## 2. ポートとURL

- **デフォルト：** `http://localhost:5173/`
- **5173・5174 が使用中の場合：** Vite は自動的に次の空きポートを使用する（例：`http://localhost:5175/`）
- ターミナルに表示された URL をブラウザで開く

---

## 3. メニュー画面の確認

以下の URL で「注文・看板管理」メニュー（Menu.vue）が表示されること。

| URL   | 挙動                         |
|-------|------------------------------|
| `/`   | `/menu` にリダイレクトされる |
| `/menu` | メニューが直接表示される     |

例：`http://localhost:5174/` または `http://localhost:5175/`、`http://localhost:5175/menu`

---

## 4. 共通スタイルが適用されているか確認（DevTools）

1. ブラウザで DevTools を開く（F12 または右クリック → 検証）
2. **Elements** タブで、共通クラスが付いた要素（例：`class="btn btn-primary"`、`class="card-header-full"`）を選択
3. **Styles** パネルに、`shared.css` や `common.css` 由来のスタイルが表示されていれば適用されている
4. ヘッダー（青背景・メインメニューリンク・ログイン表示）やボタンの見た目が意図どおりであることを確認

---

## 5. ページ別スタイルの確認

- **メニュー（/menu）：** common.css のスタイルが適用される
- **注文一覧（/order/list）：** OrderList.vue で order-list.css を読み込んでいるため、一覧専用スタイルが適用される

---

## 判定

以下が全て OK であれば、環境は正常に構築されている。

**フロントエンド**

- [ ] `npm run dev` が起動する
- [ ] 表示された localhost URL でアクセスできる
- [ ] `/` または `/menu` でメニューが表示される
- [ ] DevTools で共通スタイル（shared.css 等）が適用されていることが確認できる

**バックエンド（API 利用時）**

- [ ] `uvicorn` が起動する
- [ ] `GET http://localhost:8000/health` で `{"status":"ok"}` が返る
- [ ] `GET http://localhost:8000/companies` で DB 接続が確認できる
